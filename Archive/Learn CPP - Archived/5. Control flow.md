# switch statements
- Can only be used with integrals
```CPP
    switch (x) // x evaluates to 3
    {
    case 1:
        std::cout << "One";
        break;
    case 2:
        std::cout << "Two";
        break;
    case 3:
        std::cout << "Three"; // execution starts here
        break; // jump to the end of the switch block
    default:
        std::cout << "Unknown";
        break;
    }
```

## Fallthrough
- No break statements
- Continue falling into bottom cases
- Sometimes this is what we want -> denote with `[[fallthrough]]
```CPP
    switch (2)
    {
    case 1:
        std::cout << 1 << '\n';
        break;
    case 2:
        std::cout << 2 << '\n'; // Execution begins here
        [[fallthrough]]; // intentional fallthrough -- note the semicolon to indicate the null statement
    case 3:
        std::cout << 3 << '\n'; // This is also executed
        break;
    }
```
- Use case: sequential case labels
- eg.
```CPP
  case 'a':
  case 'e':
  ...
    return true;
```
- Scoping
	- No implicit scoping, like unlike 'if' statements
- Can declare variables inside the switch, both before and after case labels
	- All statements in switch are considered part of same scope, so can be used in later cases
		- Variable declared/defined in on e case can be used in a later case, even if the case in which variable is defined/ is never executed
		- However, **initialisation** of variables does require definition to execute
			- Thus disallowed in any case that is not last case, and before first case
			- **Allowed** in **block** in a case, since that variable is limited to that case's scope.
```CPP
switch(n) {
    case 1:
	    int x = 3; // initialiastion-not allowed since not last case.
        int y; // y exists whether or not case 1 hits. memory reserved.
        y = 5;
        break;
    case 2:
        y = 6;   // valid. assignment, not initialisation.
        break;
}
```


> If need to define and/or initialise a new varibale, best practice is to do so in explicit block underneath the case statement

# for loops
## multiple counter
```cpp
for (int x{ 0 }, y{ 9 }; x < 10; ++x, --y)
```
- Defines x and y, iterates x over 0-9 , each time increments x and decrements y
- Note the use of commas
