# For fundamental types
```cpp
int a;         // no initializer (default initialization)
int b = 5;     // initializer after equals sign (copy initialization)
int c( 6 );    // initializer in parentheses (direct initialization). NOT allowed for class member functions!

// List initialization methods (C++11)
int d { 7 };   // initializer in braces (direct list initialization)
int e = { 8 }; // initializer in braces after equals sign (copy list initialization)
int f {};      // initializer is empty braces (value initialization)
```
# For objects

```cpp
// Calls Foo() default constructor
Foo f1;           // default initialization
Foo f2{};         // value initialization (preferred)

// Calls foo(int) normal constructor
Foo f3 = 3;       // copy initialization (non-explicit constructors only)
Foo f4(4);        // direct initialization
Foo f5{ 5 };      // direct list initialization (preferred)
Foo f6 = { 6 };   // copy list initialization (non-explicit constructors only)

// Calls foo(const Foo&) copy constructor
Foo f7 = f3;      // copy initialization
Foo f8(f3);       // direct initialization
Foo f9{ f3 };     // direct list initialization (preferred)
Foo f10 = { f3 }; // copy list initialization
```
# Important points for interview
- Direct vs Copy initialisation
	- Copy initialisation cannot use explicit constructors
```cpp
Foo a = 5;   // copy initialization
Foo b(5);    // direct initialization
```

- Brace initialisation prevents narrowing
```CPP
int a = 3.5;   // allowed
int b(3.5);    // allowed
int c{3.5};    // ❌ error
```

- Most vexing parse
```cpp
Foo a(); // This is function declaration, not object instantiation
Foo a{}; // This is object

Foo b = Foo (); // This is ok.
```

- Copy constructor vs Copy initialisation
```cpp
Foo f = 5; // This does **NOT** call copy constructor.
		   // It calls Foo(int)
		   
Foo b = a; // Copy constructor called
```

- Brace initialisation prefers `initializer_list`
```cpp
class Foo {
public:
    Foo(int);
    Foo(std::initializer_list<int>);
};
Foo a(5);   // Foo(int)
Foo a{5};   // initializer_list constructor
```

- Default vs value initialisation
```cpp
// Built in
int a;   // garbage
int b{}; // 0

// Classes
Foo a;   // Default constructor called
Foo b{}; // Default constructor called
```

- Copy Elision/RVO
```cpp
Foo makeFoo() {
    Foo f;
    return f;
} // No copy happens, guaranteed copy elisiion.
```

- Pass by value causes copies (unless moved or elided)
```cpp
void func(Foo f);
func(a); // calls copy constructor
```


Answers to cj quiz
```cpp
class obj {
	obj() {cout << 1;} // normal constructor
	obj(const obj&) {cout <<2;} // copy constructor 
	void operator=(const obj&) {cout <<3;} // overloaded assignment operator
}


obj a; // default initialisation, default constructor called. 1
obj b{}; // list initialisation, default constructor called, 1.
obj c(); // nothing. most vexing parse.
obj d = a; // copy constructor, 2
d = b; // assignment, 3
auto e = obj{}; // NOT COPY! This is the same as (a). default initialisation
obj f{a}; // copy constructor, 2
obj g(a); // copy constructor, 2
auto h = b; // copy constructor, 2
obj(i); // This is the same as `obj i;`.
```
Pointers
- = assignment operator only works when the left hand side object already exists.
- Don't get confused between **copy initialisation** and **copy constructor being called**
	- eg. (e), this is just normal initialisation due to **eliding**
- `obj(i)` is the same as `obj i`
