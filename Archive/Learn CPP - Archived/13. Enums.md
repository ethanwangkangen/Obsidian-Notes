# User-defined types
- Take note: in C++, 'user defined type' is any class type/enumerated typed that is defined by you, **standard library, or implementation**

|Type|Meaning|Examples|
|---|---|---|
|Fundamental|A basic type built into the core C++ language|int, std::nullptr_t|
|Compound|A type defined in terms of other types|int&, double*, std::string, Fraction|
|User-defined|A class type or enumerated type  <br>(Includes those defined in the standard library or implementation)  <br>(In casual use, typically used to mean program-defined types)|std::string, Fraction|
|Program-defined|A class type or enumerated type  <br>(Excludes those defined in standard library or implementation)|Fraction|
- Type definitions partially exempt from ODR
	- Unlike variables and functions, forward declarations don't work for types
	- Thus, **given type allowed to be defined in multiple code files**
	- Thus, just put in headers and include in files needed.

# Enumerations
## Unscoped
```Cpp
// Color is the enumeration/enumerated type
// red is enumerator

enum Color {red, green, blue}; // Color enumeration in global scope
// Danger: red, green, blue all in global scope too, might collide with
// other enumerations in other enumerations.

Color apple {red}; 


// One solution: put in namespace
namespace Color {
	enum Color {red, green, blue};
}

Color::Color paint{Color::blue};
```
- Enumerators are compile-time constants (**constexpr**)
- Unscoped enumeration **implicitly** converts to **integral** value
	- This is a `constexpr` conversion (since they are compile-time constants)
	- Can explicitly define value of enumerator.
		- `red =4,`
		- Need **not** be unique.
- Value initialising enumeration - default 0
	- `Animal a{}` // value-initialisation zero-initialises a to default value 0
> Thus make the first enumerator the default one.

- Enumeration underlying type
	- **Which integral value to use?**
	- Can specify, but only when necessary.
	- Possible danger if using things like std::int8_t, since these are often aliases for char types
```cpp
// Use an 8-bit integer as the enum underlying type
enum Color : std::int8_t {
    black,
    red,
    blue,
};
```

#### Conversion between integer and unscope enumeration
- NO implicit conversion integer->enumeration
	- Need static_cast.
	- Range of safe casting: depends.
		- If enumeration has explicitly defined underlying type, is simply that range.
		- If not, compiler gets to pick the underlying type, so only safe to cast integral values that fit in the range of the smallest number of bits that can hold the value of all enumerators
```cpp
Pet pet { 2 }; // compile error
Pet pet { static_cast<Pet>(2) }; // ok.
```

- If base is explicitly specified, C++17 can **list initialise** unscoped enumeration with integral
```cpp

enum Pet::int {cat, dog, pig, whale} // base specified int
Pet pet1 { 2 }; // ok
Pet pet2 (2);   // compile error: cannot direct initialize with integer
Pet pet3 = 2;   // compile error: cannot copy initialize with integer
```


#### Getting name of enumerator (to print, as a string)
- Switch
``` cpp
constexpr std::string_view getColorName(Color color) {
    switch (color)
    {
    case black: return "black";
    case red:   return "red";
    case blue:  return "blue";
    default:    return "???";
    }
}
// Reminder: can return std::string_view bound to string literal, since string literal persists in memory forever.
```

#### User input to create enumerator
- cin into integer, then cast integer to enum (see above)

#### Getting enumeration from string
- Take in string, return an **std::optional**
```cpp
constexpr std::string_view getPetName(Pet pet) {
    switch (pet)
    {
    case cat:   return "cat";
    case dog:   return "dog";
    case pig:   return "pig";
    case whale: return "whale";
    default:    return "???";
    }
}

constexpr std::optional<Pet> getPetFromString(std::string_view sv) {
    // We can only switch on an integral value (or enum), not a string
    // so we have to use if-statements here
    if (sv == "cat")   return cat;
    if (sv == "dog")   return dog;
    if (sv == "pig")   return pig;
    if (sv == "whale") return whale;

    return {};
}

int main() {
    std::cout << "Enter a pet: cat, dog, pig, or whale: ";
    std::string s{};
    std::cin >> s;

    std::optional<Pet> pet { getPetFromString(s) };

    if (!pet)
        std::cout << "You entered an invalid pet\n";
    else
        std::cout << "You entered: " << getPetName(*pet) << '\n';

    return 0;
}
```

### Operator overloading
- Overloading `operator<<` to print an enumerator
	- `std::cout << 5` 
		- `std::cout`  has type `std::ostream`
		- `5` is literal of type `int`
```cpp
std::ostream& operator<<(std::ostream& out, Color color)
{
    out << getColorName(color); // print our color's name to whatever output stream out
    return out;                 // operator<< conventionally returns its left operand

    // The above can be condensed to the following single line:
    // return out << getColorName(color)
}
```
- Overloading `operator>>` to input enumeration
```cpp
// pet is an in/out parameter
std::istream& operator>>(std::istream& in, Pet& pet)
{
    std::string s{};
    in >> s; // get input string from user

    std::optional<Pet> match { getPetFromString(s) };
    if (match) // if we found a match
    {
        pet = *match; // dereference std::optional to get matching enumerator
        return in;
    }

    // We didn't find a match, so input must have been invalid
    // so we will set input stream to fail state
    in.setstate(std::ios_base::failbit);

    // On an extraction failure, operator>> zero-initializes fundamental types
    // Uncomment the following line to make this operator do the same thing
    // pet = {};

    return in;
}

int main()
{
    std::cout << "Enter a pet: cat, dog, pig, or whale: ";
    Pet pet{};
    std::cin >> pet;

    if (std::cin) // if we found a match
        std::cout << "You chose: " << getPetName(pet) << '\n';
    else
    {
        std::cin.clear(); // reset the input stream to good
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
        std::cout << "Your pet was not valid\n";
    }

    return 0;
}
```

## Scoped Enum (Enum classes)
```cpp
enum class Color{ //keyword 'class'
    red, // red is considered part of Color's scope region
    blue,
};

std::cout << Color::red;
```

- No implicit conversion to int, but can explicitly convert
```
std::cout << static_cast<int>(color) << '\n'; 
```
- More advanced way, overload the `operator+` to convert to int.
```CPP
template <typename T>
constexpr auto operator+(T a) noexcept
{
    return static_cast<std::underlying_type_t<T>>(a);
}
```


- Can cast integer to scoped enumerator, useful when taking in input
```CPP
Pet pet{ static_cast<Pet>(input) }; // static_cast our integer to a Pet
```
