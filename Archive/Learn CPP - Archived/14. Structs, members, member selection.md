# Structs
```cpp
struct Employee {
    int id {}; // {} initialisation ensures member variables are value intiialised when Employee created
    int age {};
    double wage {};
};

Employee joe {};
```

- Member - variable/function/type that belongs to a struct (or class).
	- All members must be declared within the struct/class function
- Member selector operator
	- `joe.age`

## Aggregate data types
- C-style array, or
- Class type (struct, class, union) that has
	- No user declared constructrs
	- No private/protected non-static data members
	- No virtual functions

## Initialisation
- **Non-static member initialisation**
	- Can provide **default** initialisation value for each member, directly inside class/struct defition
	- `int age{2};`
	- Value initialisation
		- `int age{};` -> default value

> Use list initialisation to default initialise.

- **Aggregate** initialisation
	- `Employee frank = { 1, 32, 60000.0 };`
	- **Initialiser list** as initialiser
	- **Memberwise initialisation**
		- Each member initialised in order of declaration
			- `id` first, then `age`,then `wage`

- If have explicit initalisers, ie. no missing, then all explicit values used (no default values used)
- Missing initialisers in initialiser list (fewer given)
	- If the member has a default member initializer, that is used
	- If not, member is copy-initialized from an **empty initializer list**

> This means we can use empty initialisation list to value initialise all members of struct
> `Employee joe {};`

- Note that both `Employee joe;` and `Employee joe {};` allowed, but only the latter will value initialise all members, **even in the case that default values not given** .
	- ie. `int age;` -> missing default values.
	- `Employee joe;` -> age is not value initialised, so undefined
	- `Employee joe{}` -> age is copy initialised from empty intialiser list, ie 0.

> Advised to use `Employee joe {};` over `Employe joe;`
## Overloading operator<< to print 
```cpp
std::ostream& operator<<(std::ostream& out, const Employee& e){
    out << e.id << ' ' << e.age << ' ' << e.wage;
    return out;
}
```

## Const structs
- Variables of struct type can be const or constexpr
- Like all const variables, must be initialised
```
const Rectangle unit { 1.0, 1.0 }
```

## Assignment
- Can do it with initialiser list
	- `joe = { joe.id, 33, 66000.0 }; //update values all at once`

## Passing structs
- Usually by **const ref**
	- `void function(const Employee& employee)`
- Can pass in temporary objects
	- `printEmployee(Employee { 14, 32, 24.15 });` explicit typing
	- `printEmployee({14, 32, 24.15})` type deduced from parameter

## Returning structs
- Usually return by value
- Type return deduction
	- SImilar to parameter type deduction (above)
	- `return {0, 1, 2};`

## Structs within program-defined members
```CPP
struct Company {
	Employee CEO{};
}
```

## Nested structs
- Can nest enum within struct too. Eg. enum Type{...} within struct Monster{..}
```cpp
struct Company {
    struct Employee { // accessed via Company::Employee
        int id{};
        int age{};
        double wage{};
    };
    int numberOfEmployees{};
    Employee CEO{}; // Employee is a struct within the Company struct
};
```

## Ownership
- Structs that are owners should have data members that are owners
- Similar concept to `std::string` and `std::string_view`

## Struct size and data alignment
- At least as large as size of all variables it contains
	- But could be large!
		- Padding
	- Reduce this by defining members in decreasing order of size
	- Keep in mind, C++ does not reorder members.

## Member selection from pointer operator (->)
```CPP
Employee joe {1,2,3};
Employee* ptr {&joe};

(*ptr).age +=1;
// is equivalent to
ptr->age +=1;
```
