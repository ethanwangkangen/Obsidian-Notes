# std::optional
- Some functions can't reasonably handle errors, eg. functions that return results should have no side effects
	- Previously, saw 2 ways to do this
		- Have function return bool indicating success/failure
		- Return sentinel value (special value) to specify error
- C++17: `std::optional`
	- `std::optional<T>` can either have value of `T` or not

```CPP
// Initialising
std::optional<int> o1 { 5 };            // initialize with a value
std::optional<int> o2 {};               // initialize with no value
std::optional<int> o3 { std::nullopt }; // initialize with no value

// Checking value
if (o1.has_value()) // call has_value() to check if o1 has a value
if (o2)             // use implicit conversion to bool to check if o2 has a value

// Getting data
std::cout << *o1;  // dereference to get value. undefined if no value
std::cout << o2.value(); //throws std::bad_optional_access if no value
std::cout << o3.value_or(42); //get value or 42 if no value

// Example
std::optional<int> doIntDivision(int x, int y)
{
    if (y == 0)
        return {}; // or return std::nullopt
    return x / y;
}
```

## Using std::optional as optional function parameter
- Instead of using **default parameters** - see [[8. Functions - overloading, deleting, templates]]
	- ie. `void printIdNumber(const id* ptr = nullptr) {..}`
- eg
```cpp
void printIDNumber(std::optional<const int> id = std::nullopt) {
    if (id)
        std::cout << "Your ID number is " << *id << ".\n";
    else
        std::cout << "Your ID number is not known.\n";
}
```
- However, requires copy initialisation as `std::optional<T>` does not support references
	- If expensive copy, use `const T*`

> The better way to do this is overloading function.
