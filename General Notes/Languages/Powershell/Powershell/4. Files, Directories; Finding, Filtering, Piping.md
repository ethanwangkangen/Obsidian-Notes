eg. `Get-ChildItem test
- Depends on what `test` is
	- Folder named test → lists the contents of that folder (files/subfolders in .\test\).
	- File named test → returns that single file object.
	- Pattern (PowerShell does globbing) → e.g. test* matches anything starting with “test”.
- Output is objects (FileInfo / DirectoryInfo), not plain text—so you can pipe to other commands
	- (e.g. `| Select-Object Name, Length`).
- `-recurse` for recursive search down directories

# Piping
- Similar idea as in Linux, but what flows through the pipe is **.NET objects** instead of just bytes
- Output of one command goes into input of the next
- `&&` : Next command only if current one passes
- `||`: Next command only if current one fails
# Finding and filtering
- In PowerShell, everything is an **Object**
	- **Object** is an instance of a **Class** (OOP concepts apply)
	- Has
		- **Properties** -> data
		- **Methods** -> functions

## Listing
- `Get-Member`
	- List members of an object (both properties and method)
## Reshaping
- `Select-Object -Property Name
	- Only show specified properties for objects

## Filtering
- `Where-Object <condition>`
	- alias: `where`
	- Filters for objects where condition evaluates to treu.
	- eg.
		- `Get-Process | Where-Object { $_.CPU -gt 5 -and $_.ProcessName -like 'chrome*' }
			- `{ ... }` is a script block. Code inside it evaluated, treated as `$true/$false`, then used for Where-Object to filter
		- `Get-ChildItem | Where-Object Length -gt 1MB`
		- `Get-Service | where DisplayName -like '*Defender*'`
			- This, for eg., is actually bound to parameterised form 
			- `Get-Service | where -Property DisplayName -Like -Value '*Defender'
			- which evalautes like `Get-Service | where {$_.Display-Name -like '*Defender'}`
		- `Get-Child_Item | Where-Object { $_.Length -gt 1MB -and $_.Name -like '*.log' }
		- `Get-Childitem –Path C:\Users\MTanaka\ -File -Recurse -ErrorAction SilentlyContinue | where {($_.Name -like "*.txt" -or $_.Name -like "*.py" -or $_.Name -like "*.ps1" -or $_.Name -like "*.md" -or $_.Name -like "*.csv")}

#### Comparison Operators

| **Expression** | **Description**                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Like`         | Like utilizes wildcard expressions to perform matching. For example, `'*Defender*'` would match anything with the word Defender somewhere in the value. |
| `Contains`     | Contains will get the object if any item in the property value matches exactly as specified.                                                            |
| `Equal` to     | Specifies an exact match (case sensitive) to the property value supplied.                                                                               |
| `Match`        | Is a regular expression match to the value supplied.                                                                                                    |
| `Not`          | specifies a match if the property is `blank` or does not exist. It will also match `$False`                                                             |
