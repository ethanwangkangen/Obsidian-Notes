# Decay
## Functions decay - to function pointers
- Functions decay to pointers, just like arrays do
	- Function **name** decays to **pointer to itself**
```cpp
void(*fp1)(int) = func;
void(*fp2)(int) = &func;

// calling through the pointer — these are ALL identical:
fp1(42);
(*fp1)(42);
(**fp1)(42);
(***fp1)(42);  // yes, this works
```

Where decay DOESN"T happen
```cpp
// 1. decltype
void func(int);
using T1 = decltype(func);   // T1 = void(int)      — function TYPE, not pointer
using T2 = decltype(&func);  // T2 = void(*)(int)    — pointer

// 2. Template reference parameters
template <typename F>
void take_ref(F& f);

take_ref(func);   // F = void(int), parameter is void(&)(int) — function REFERENCE
```
## **STATELESS** Lambdas decay - to function pointers
- If lambda is **captureless**, it gets a free conversion to function pointer
```cpp
void register_callback(void(*callback)(int)) {
    callback(42);
}

int main() {
    register_callback([](int i){ std::cout << i; });  // implicit conversion
}
```
# Functors
```cpp
struct FuncObj {
    void operator()(int i) const { std::cout << i; }
};
```
- Any class/**struct** with the `operator()` defined
- Passing functors as argument
	- Same way you would pass in a struct.




# Quiz
```cpp
namespace getcracked {
    template <typename Begin, typename End, typename Pred>
    void for_each(Begin begin, End end, Pred predicate) {
        if (begin > end) {
            throw std::logic_error("Begin cannot be past end.");
        }

        for (; begin != end; begin++) {
            predicate(*begin);
        }
    }
}


void func(int i) { std::cout << i; }
struct FuncObj 
{ 
    void operator()(int i) const { std::cout << i; }
};


int main() {
    std::vector<int> vec{ 1, 2 };
    getcracked::for_each(vec.begin(), vec.end(), func); // 1
    getcracked::for_each(vec.begin(), vec.end(), &func); // 2
    getcracked::for_each(vec.begin(), vec.end(), &FuncObj()); // 3
    getcracked::for_each(vec.begin(), vec.end(), FuncObj()); // 4
}
```
1-> ok. Function pointer
2-> ok. Function pointer. 
3-> NOT ok. Cannot take the address of an rvalue, which is what FuncObj() does.
4-> ok. Can pass in functor as rvalue.
