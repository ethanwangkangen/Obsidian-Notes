# Function signature
- Name
- Parameters
	- Number
	- Types
- Function level qualifiers (const, ref qualifiers (&, &&)..)
- **NOT return type!!!**

# Method overloading
- Same name, different parameter list
	- **return type does not matter**
	- `const` on parameters does not count for overloading, **UNLESS it's for REFERENCES or POINTERS**
	- `const` on member function types **does** count

```cpp
// const member functions
struct S {
    void f() const;  // const member
    void f();        // non-const member
};

S s;
s.f(); // calls non-const version
const S cs;
cs.f(); // calls const version
```

```cpp
// const parameter overloads

// 1. top level const - ambiguous overload
void f(int);  
void f(const int);
int x = 5;  
f(x); // ambiguous.

// 2. low level const (reference) 
void f(int& x);  
void f(const int& x);
int a = 5;  
const int b = 10;  
f(a); // calls int&  
f(b); // calls const int&

// 3. low level const (pointers)
void f(int* x);
void f(const int* x); // pointer to const - distinct overload YES
void f(int* const x); // const pointer - distinct overload NO
```
- Top level consts are ignored, but bottom level consts do count for overloads
	- **Top**-level consts apply to the **object** itself, eg. const **variables**/const **pointers**
	- **Bottom**-level consts apply to the **pointee** or **referenced object**

## Reference types
- `int&` binds only to lvalues.
- `const int&` can bind to rvalues.
```cpp
void g(int& x);
void g(const int& x);

int a = 5;
g(a);   // calls g(int&)
g(10);  // calls g(const int&)
```

## Pointers and derived classes
- The **most derived**/ exact match preferred
- Implicit conversions used only if no match exists

```cpp
struct Base {};
struct Derived : Base {};

void h(Base* b);
void h(Derived* d);

Derived d;
h(&d); // calls h(Derived*), exact match preferred
```

- Rules (in order of hierarchy)
	- **Exact** match: argument types match paramter types exactly
	- **Promotion**: smaller integer types promoted (char->int)
	- **Standard** **conversion**: numeric converison, pointer conversions, derived->base pointer..
	- **User-defined conversion**: calls a constructor or conversion operator
	- **Ellipsis**: ... matches anything, lowest priority
- Note: if multiple matches in one step, ambiguous function call will result.
	- Note: int to unsigned int also considered a conversion. So

```cpp
	foo(unsigned int) {..}
	foo(float) {..}
	foo(3.145); //ambiguous! can convert to either one
	foo(0); //ambiguous! can convert to either one
```

- Default arguments can complicate things
```
void f(int x);
void f(int x, int y = 0);

f(5); // ❌ Ambiguous: could call either
```

- When multiple arguments
	- Rules applied to each argument
	- Function chosen is the one for which each argument matches at least as well as all other functions with at least one matching better than all the others
		- Better as in less steps needed
