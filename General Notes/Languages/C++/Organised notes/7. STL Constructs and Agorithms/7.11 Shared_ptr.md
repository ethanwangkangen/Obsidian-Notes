# Deleters
- Deleters are **type-erased**
- Unlike in unique_ptr,
	- **NOT part of the type**
	- 2 shared_ptr of the same type can have different deleters, can be stored in same container
- The deleter lives on the **heap**, **inside the control block**
- Can take in anything callable with the right signature
	- Lambda
	- Function pointer
	- Functor/function object
	- Function

```cpp
// Lambda
std::shared_ptr<FILE> p1(fopen("a.txt", "r"), [](FILE* f) { fclose(f); });

// Function pointer
void close_file(FILE* f) { fclose(f); }
std::shared_ptr<FILE> p2(fopen("a.txt", "r"), close_file);

// Functor / function object
struct FileCloser {
    void operator()(FILE* f) const { fclose(f); }
};
std::shared_ptr<FILE> p3(fopen("a.txt", "r"), FileCloser{});

// std::function
std::function<void(FILE*)> del = [](FILE* f) { fclose(f); };
std::shared_ptr<FILE> p4(fopen("a.txt", "r"), del);
```

## When necessary to use custom deleters?
- Eg. `array`

```cpp
std::shared_ptr<Dog> p (new Dog[3]); // Deleter will only delete first dog.

std::shared_ptr<Dog> p (new Dog[3], [](Dog* p) {delete[] p;});
```
- Default destructor will call `delete p`, which only deletes the first Dog
- Need to specify array delete `delete[]` through custom deleter

# std::enable_shared_from_this
- When an object managed by `shared_ptr` needs to obtain a `shared_ptr` to itself
	- Can't just do `shared_ptr<MyClass>(this)` -> will result in new control blocks each time, and double-delete
```cpp
struct Server : std::enable_shared_from_this<Server> {
    void start_async_op() {
        // Capture a shared_ptr to this, keeping the object alive
        // until the callback completes
        auto self = shared_from_this();
        post_callback([self]() {
            self->handle_result();
        });
    }
};

// Must be owned by a shared_ptr first:
auto s = std::make_shared<Server>();
s->start_async_op();
```
