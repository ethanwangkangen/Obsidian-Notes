# Iterators
- Different STL constructs have their own iterator types
```cpp
std::vector<int>::iterator
std::list<int>::iterator
std::map<int,int>::iterator
```
- Therse are all unrelated types
- Do not inherit from same Base class, no virtual functions, no runtime polymorphism
- However, they do **implement the same functionality**
	- Interface is enforeced by requirements, not inheritance

## Functionality
- `*it` (dereference)
- `++it` (increment)
- `it != end`
Formally, it must satisfy a **concept** (C++20)

|Concept|Requirements|
|---|---|
|InputIterator|read, increment|
|ForwardIterator|multi-pass|
|BidirectionalIterator|++ and --|
|RandomAccessIterator|arithmetic + indexing|

# STL Algorithms
```cpp
template <typename It>
void sort(It first, It last);
```
- They use **iterators**, but do not care about what `It` is
- Only requires that they support the operations shown above
	- If those operations exist and behave correctly, it compiles
- This is called **duck typing**
	- If it looks and behaves like an iterator, it is an iterator
