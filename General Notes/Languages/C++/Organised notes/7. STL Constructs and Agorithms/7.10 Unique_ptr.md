namespace getcracked 
{
    template <typename T>
    struct custom_deleter {
        void operator()(T* pointer) const {
            delete pointer;
        }
    };

    template <typename T, typename Deleter = custom_deleter<T>> 
	    // custom_deleter -> default template argument
    class unique_ptr {
    public:
        unique_ptr() { 
        }

        unique_ptr(T* pointer) {
            myPtr = pointer;
        }
        
        // copy constructor throws
        unique_ptr(const unique_ptr&) { 
	        throw  std::runtime_error("Copy constructor called"); 
	    }

        // copy assignment throws
        unique_ptr& operator=(const unique_ptr&) { 
	        throw  std::runtime_error("Copy assignment called");  
        }

        // move constructor
        unique_ptr(unique_ptr&& other) noexcept : myPtr{other.release()} { }
		
		// move assignment
        unique_ptr& operator=(unique_ptr&& other) noexcept {
            if (&other == this) return *this;
            myPtr = other.release();
            return *this;
        }

        ~unique_ptr() {
            Deleter myDeleter{};
            myDeleter(myPtr);
        }

        T* release() {
            T* temp = myPtr;
            myPtr = nullptr;
            return temp;
        }

        void reset(T* pointer) {
	        Deleter myDeleter{};
            myDeleter(myPtr);
            myPtr = pointer;
        }

        bool is_owning() const {
            return myPtr != nullptr;
        }

        T& operator*() const {
            return *myPtr;
        }
        T* operator->() const {
            return myPtr;
        }

        operator bool() const {
            return is_owning();
        }

    private:
        T* myPtr{nullptr};
        
    };
}

- Mistakes made
	- Not doing/knowing how to do self assignment check for move assignment
		- We are checking if the **address** is the same
		- `&&other`is a reference -> use `&`to get address of `other`
		- `this` is a void pointer -> **already** holds address
	- Not returning `this` for move assignment
	- Not knowing how to use **custom deleter**
		- Making custom deleter a member -> no! Just instantiate it at the point of deletion (in the destructor)
	- Mistakes in move constructor
		- Forgetting that **noexcept** has to come **before** the member initialiser list


# Deleter
- `std::unique_ptr<T, Deleter>` accepts any type as custom deleter as long as it's **callable with a pointer of type T**** 
	- Function pointer
		- `void (*) (T*)`
		- `void myDelete(Foo* p) {... delete p;}`
		- `std::unique_ptr<Foo, decltype(&myDelete)> ptr(new Foo, myDelete);
	- Function objects
		- Any struct with `operator() (T*)
		- `struct MyDeleter{ void operator() (Foo* p) const {delete p;} };`
		- `std::unique_ptr<Foo, MyDeleter> ptr(new Foo);`
			- Default constructible (is a struct), so no need to pass in an instance
	- Lambdas
		- Using `decltype` or wrapping in `std::function`
		- `auto del = [](Foo* p) { delete p; };` 
		- `std::unique_ptr<Foo, decltype(del)> ptr(new Foo, del);`
		- `std::unique_ptr<Foo, std::function<void(Foo*)>> ptr(new Foo, [](Foo* p) { delete p; });

- Things to note
	- Function pointers and `std::function` **increases the size** of `unique_ptr` (typically up to 2 pointers) 
		- while empty/stateless functors and captureless lambdas benefit from **empty base optimisation**, keeping size of unique_ptr to single pointer

- How to know if size is increased?
	- Whether or not the deleter is **stateful** -> does it have **data members**?
		- Empty type (no data members) -> zero overhead
		- Non-empty type (has data members) -> adds to size
			- Function pointer stores a pointer
			- std::function stores a type-erased callable
			- Lambda with captures stores captured state
			- Functor with data members
	- NOT whether or not an instance is passed in (misconception)
		- If struct with no data members is passed in as instance, still no additional overhead

- C++ 20
	- Can use stateless lambdas directly since they're **default-constructible**
	- `std::unique_ptr<Foo, decltype([](Foo* p) { delete p; })> ptr(new Foo);`
