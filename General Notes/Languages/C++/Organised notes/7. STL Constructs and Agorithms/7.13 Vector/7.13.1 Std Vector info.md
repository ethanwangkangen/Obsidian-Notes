# List constructors
```cpp
std::vector<int> primes {1,2,3,4,5}; // list constructor
std::vector<int> things {1,2}; // list constructor, 2 elements
std::vector<int> fiveElements (5,2); // 2 argument constructor, 5 elements of 2
std::vector<int> fourElements (4); // single argument constructor, 4 elements of 0
std::vector<int> oneElement {3}; // list constructor, 1 element
```
- When constructing class type object with initialiser list
	- If initialiser list empty, **default constructor preferred**
	- If initialiser list non-empty, **matching list constructor** preferred over others
		- Hence `vector<int> oneElement{3}` uses list constructor

# const vector
```cpp
const std::vector<int> prime {1,3};
```
- Cannot be modified

# no constexpr vector
- Use `std::array` instead

# length and indices type - size_type
- Lengths and indices are type `size_type, which defaults to `std::size_t` (alias for it)
	- Is **unsigned**
- All STL containers except `std::array` use `std::allocator` to allocate memory
	- And for these, `T::size_type` is derived from the `size_type` of the allocator used
		- Which is usually `std::size_t`

- Length
	- `std::size(prime)`
	- `prime.size()`
	- To convert to int, just `static_cast<int>`

- Indexing
	- Need to cast to `size_t`, **UNLESS** we use a constexpr (signed) int, in which case compiler can implicitly convert without narrowing

# Passing vectors
- Different types -> function templates
```cpp
template <typename T>
void passByRef(const std::vector<T>& arr) {
    std::cout << arr[0] << '\n';
}
```

- Generic template
```cpp
template <typename T>
void passByRef(const T& arr) { // will accept any type of object that has an overloaded operator[]
    std::cout << arr[0] << '\n';
}
```
- If pass in a non-subscriptable type here,
	- Template instantiation fails at compile time, as it tries to **instantiate the body**, but will give cryptic error
	- Better way -> SFINAE method / Concepts in C++ 20 (KIV) this

# Passing vectors
- Pass by const reference
- If want to give up ownership, can move instead
	- 2 options
1. Pass by rvalue reference
```cpp
void takeOwnership(std::vector<int>&& v) {
    std::vector<int> myVec = std::move(v);  // must still move explicitly inside
    // ...
}

std::vector<int> big(1'000'000, 42);
takeOwnership(std::move(big));  // caller explicitly gives up ownership
// big is now in a valid but unspecified state — don't use it
```
OR
2. Pass by value (sink parameter)
- This is the standard idiom for "this function takes ownership"
	- Caller has choice - can either move OR copy in
	- Simpler to write (one overload instead of potentially needing both const& and && overloads)
	- Extra move (into the parameter) is cheap for move-capable types
```cpp
void takeOwnership(std::vector<int> v) {
    // v is already your own copy — no further move needed
    // ...
}

takeOwnership(std::move(big));  // moves into the parameter
takeOwnership(someTempVec());   // moves from the temporary
takeOwnership(big);             // copies if you don't move — caller's choice
```
# Returning vectors
- Retruning from a function, compiler does either of the follwing
	- NRVO: compiler constructs the vector directly in caller's memory. No copy, no move..
	- NRVO not applied: returned local is treated as rvalue, so **move constructor** kicks in. Only need to copy 3 pointers (data, size, capacity) and null the source.
```cpp
std::vector<int> makeVec() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    return v;  // NRVO or move — either way, cheap
}

auto result = makeVec();  // no expensive copy ever happens
```
