# std::vector vs std::array
- **`std::array<T, N>`**
    - Fixed-size array wrapper (size known at compile time)
    - Stack-allocated (usually)
    - No dynamic resizing
- **`std::vector<T>`**
    - Dynamic array
    - Heap-allocated storage
    - Can grow/shrink at runtime

## Performance considerations

- **`std::array`**
    - Faster access (no indirection, no heap)
    - No allocation cost
    - Best cache predictability
- **`std::vector`**
    - Slight overhead due to heap allocation
    - Amortized O(1) `push_back`
    - Possible reallocations → invalidates pointers/references/iterators

# Std::vector implementation
```cpp
template<typename T>
struct vector {
    T* first; // beginning of single memory allocation
    T* last; // pointer one past the end of last valid
    T* end; // pointer one past the end of the last allocated mem for vector
};
```
- size() is last-first
- capacity() is end-first

For interviews, 
```cpp
template<typename T>
struct vector {
    T* data;
	size_t size;
	size_t capacity;
};
```
