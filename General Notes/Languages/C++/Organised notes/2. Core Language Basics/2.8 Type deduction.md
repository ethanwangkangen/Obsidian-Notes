# Type deduction/inference
- `auto`
- eg.
	- `auto d { 5.0 }; // 5.0 is a double literal, so d will be deduced as a double`
- Note: prior to C++17, `{}` initialisation will deduce type to be that osf `std::initializer_list<>` instead.
- Must have something to deduce from, ie. need to copy/direct initialise, not just declare

## const
- `const` **dropped**
	- **To keep, use `const auto`**
	- Top level const dropped - refers to the object itself
		- `const int x`
		- `int* const ptr` (pointer it constant)
	- Low level kept - refers to the object being referenced to
		- `const int& ref`
		- `const int* ptr` = `int const* p`
-  `constexpr
	- not even part of the type system. 
	- implicitly const, but const dropped (see above)
	- can put it back with `constexpr`

## References
- Reference **dropped**
	- Just like `const` is dropped 
	- Can explicitly state that its a reference
```cpp
std::string& getRef(); //fn that returns ref

auto ref {getRef()}; // const dropped, deduced to std::string and not std::string&

auto& ref {getRef()}; // explicitly state reference
```

- If initialiser is reference to const, reference is dropped first (then reapplied if applicable), then top-level const is dropped from result.
	- Dropping reference may change a low level const to a top level const
	- However, if reference is reapplyed, dropping the const may not apply
eg.
```cpp
const std::string& ref = str;
auto x = ref; //x is std::string& (ref dropped, then const dropped)

auto& x = ref; //x is const std::string& (ref reapplied, so const kept)
```
- **const** is preserved here!!

> Best practice: specify `const` and `&` explicitly when using `auto`
## String literals
- String literals
```CPP
auto s { "Hello, world" }; // s will be type const char*, not std::string
auto s1 { "goo"s };  // std::string literal
auto s2 { "moo"sv }; // std::string_view
```


> Use type deduction for variables when type of object doesn't matter.
> Favor explicit type when require a specific type that differs from initialiser, or when object is used in context where seeing the type is useful

## Functions
```CPP
auto add(int x, int y) {
    return x + y;
} // return statement returns int, so function deduces int return
```
- Forward declaration insufficient. Must be fully defined before use

> Try to avoid type deduction for functions

- Cannot use in parameters.
	- Works in C++20, but is not invoking type deduction, instead is using *function templates*

## Pointers
- **NOT dropped**
```CPP
auto ptr1{ getPtr() };  // std::string*
auto* ptr2{ getPtr() }; // std::string*
```

When does `auto` and `auto*` differ?
- When `const` pointers.

```CPP
// No difference here.
const auto ptr1{ getPtr() };  // std::string* const
auto const ptr2 { getPtr() }; // std::string* const

// Different!
const auto* ptr3{ getPtr() }; // const std::string* (pointer to const)
auto* const ptr4{ getPtr() }; // std::string* const (const pointer)
```

> If `const` comes before the `*` sign, its a pointer to constant. Object is constant.

> Specify `auto*` when deducing pointer type

### Tip
- Read right to left. (simple heuristic, doesn't always apply)
- eg.
```cpp
int* const ptr
// ptr → const pointer → to int

const int* ptr
// ptr → pointer → to const int

const int& r
// reference -> to const int
```

### Proper way to read
- Start at identifier
- Look right first
- Then left
- Respect parenthese
- eg.
```cpp
int* p[5]; // array[5] of pointers to int
int (*p)[5] //pointer to array[5] of int
const int* const p // const pointer to const int
int const* p //pointer to const int
```

# auto&&
- Behaves like template `T&&` (see section on Templates later on)
- Takes the value, detect if it's rvalue or lvalue, and **attach the appropriate reference**

| original type | auto deduction | example                                      |
| ------------- | -------------- | -------------------------------------------- |
| lvalue        | int&           | `int a = 10;`<br>`auto&& x = a; //x is int&` |
| rvalue        | int&&          | `auto&& x = 10; //x is int&&;`               |

# decltype
- Gives the **exact type** of the **expression**, including **references** and **const**
	- Not of the **variable/value**. Of the EXPRESSION
```cpp
int x = 10;
int& ref = x;

decltype(x) a = 5;    // int
decltype(ref) b = x;  // int&

// TRICK!
int x = 10;  
decltype(x) a = x; // int  
decltype((x)) b = x; // int& - since (x) is expression, treated as lvalue
```

|Expression type|decltype result|
|---|---|
|variable `x`|declared type|
|lvalue expr|`T&`|
|rvalue expr|`T`|

# Typedef and aliases
- Alias
	- eg `using Distance = double;`
	- Then can use `Distance` in future
		- `Distance miles{1.4};`
	- Follow same scoping rules as variable identifiers
		- If inside block, block scope. If in global namespace, has global scope
		- If need type aliases across multiple files, can be defined in header file and `#included` into any files that need it
- Typedef
	- `typedef double Distance;`
	- Same use as alias

> Prefer alias to typedef
