# std::function
```cpp
std::function<ReturnType(argType1, argType2, ..)> f;

// eg.
std::function<int(int, int)> f;
```
- Holds, copy and invoke any callable

```cpp
// 1. Regular function
int add(int a, int b) { return a + b; }
std::function<int(int,int)> f = add;
```

```cpp
// 2. Lambda
std::function<int(int,int)> f = [](int a, int b) { return a + b; };
```

```cpp
// 3. Functor (object with operator())
struct Adder {
    int operator()(int a, int b) { return a + b; }
};
std::function<int(int,int)> f = Adder{};
```

```cpp
// 4. Member function (needs std::bind or lambda wrapper)
struct Foo {
    int multiply(int a, int b) { return a * b; }
};
Foo foo;
std::function<int(int,int)> f = [&foo](int a, int b) { return foo.multiply(a, b); };
```

```cpp
// 5. std::bind
std::function<int(int)> f = std::bind(add, 10, std::placeholders::_1);
f(5); // returns 15
```

- Problem that this solves
	- Can't store lambdas in a vector of `auto`s
	- **Type erasure**
		- `std::function` hides the underlying callable type

- Cost
	- Holds a pointer to a heap allocated wrapper, which itself stores a vtable pointer.
		- The vtable then points to the specific lambda's `invoke`, `clone` and `destrucotr`
	- May heap allocate to store the callable
	- Virtual dispatch on every invocation
	- Cannot be inlined
```cpp
┌─────────────────────────────────────────┐
│    std::function<int(int)>              │
│                                         │
│  ┌─────────────┐    ┌────────────────┐  │
│  │ type-erased │    │ heap-allocated │  │
│  │  vtable ptr │───▶│  callable copy │  │
│  └─────────────┘    └────────────────┘  │
│                                         │
│  + overhead on every call()             │
└─────────────────────────────────────────┘
```
