# Static local variables (in functions)

- Persists across calls
- Have static storage duration, so **zero intialised**
```cpp
void foo() {
	static int count{};
	count++;
}
```

# Static member variables
- Does not belong to the class instance
- Have static storage duration, so **zero intialised**
- **Without `inline`**, a static member declaration **inside a class body** is **always** just a **declaration**, **NOT A DEFINITION**
	- Rationale: ODR -> if a header is included in 5 `.cpp` files, you'd get 5 copies, linker doesn't know which is canonical
- With `inline`, compiler now has permission to deduplicate
	- No longer a reason to prohibit the definition from living inside the class
	- So standard allows `inline static` declarations to also be definitions

```cpp
static int x;          
// "This exists, but I can't define it here because 
//  multiple TUs might include this and I'd get duplicates."

inline static int x;   
// "This exists, define it here, and if multiple TUs 
//  see this, just keep one copy."
```

Example in use case
- Leetcode questions
	- Without the `inline` keyword, this doesnt compile
```cpp
class Solution {
public:
    inline static int memo[100][100][200];
    bool hasValidPath(vector<vector<char>>& grid) {
        memset(memo, 0, sizeof(memo));
        return dfs(grid, 0, 0, 0);
    }
    //...
```

# Static member functions
- No `this` pointer
```cpp
class Math {
public:
    static int add(int a, int b) { return a + b; }
};
Math::add(2, 3);  // no object needed
```

## Static lambdas (C++23)
- Similar to above
	- The lambda's `operator()` becomes a static member functino of the closure type
	- Result: **cannot capture anything**
		- Good to communicate intent, that this lambnda is pure functino with **no state**
	- Note: without `static`, lambda still captures the `this` pointer to closure object, which is technically unnecessary overhead
```cpp
auto add = [](int a, int b) static { return a + b; };
```

# Static at file/namespace scope
- Internal linkage
	- This is the tricky one, different from all the above
- Comprises **functions AND variables**
```cpp
static int secret = 42;
static void helper() {..}
```
