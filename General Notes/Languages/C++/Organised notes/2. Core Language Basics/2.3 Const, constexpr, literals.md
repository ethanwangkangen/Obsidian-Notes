# const
- Promise not to change this value once assigned
- **const objects**
	- Immutability guaranteed
		- For member functions, can only use **const** member functions
- Must be initialised
	- For class members, need to be initialised in **member initialiser list**, cannot use constructor (since this is assignment)!
- Const is part of object's **type**

## const references
- `const T&`
	- This **extends lifetime** of the temporary it binds to!
	- If rvalue passed in, temporary object created via copy, then lifetime of that object is extended to lifetime of the reference

## pointers
- `const char* p` -> this is a pointer to a const
	- ie. cannot modify the thing that the pointer points to
	- BUT **can reseat the pointer**
- `char* const p` -> const pointer
	- cannot reseat the pointer

> Read declarations right to left

# constexpr
- **Implicitly const**
- Is not part of the object's type
	- So type becomes **const**
- Value is known and fixed at compile time
	- Can be used in contexts that require compile-time constants
- CAN be evaluated at compile time, but may not (compiler chooses)

- constexpr variables - requirements
	- type has to be a literal (see definition below)
	- type has a `constexpr` constructor (for classes/structs)
	- all constructor arguments are constexpr
	- all members of the type are **literal types**

- literal
	- built-in literals: bool, int, double etc.
	- structs, where all members are literal types
	- value can exist entirely in compile-time evaluation

- constexpr functions [[3.3 Constexpr functions]]
	- return value can be computed at compile time
	- C++ version dependent
		- C++ 14 onwards: can have branches, loops
	- Can be used in constant expression

> Use `constexpr` whenever the value is known at compile-time and could be used in compile-time contexts. 
> Use `const` if it can’t be known at compile-time or depends on runtime computation

|---|---|---|---|

|boolean value|true, false|bool||
