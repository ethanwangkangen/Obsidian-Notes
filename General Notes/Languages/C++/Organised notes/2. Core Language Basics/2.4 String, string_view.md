## C- style string literals
- eg. "Hello world"
	- Type: `const char[14]
	- Has implicit null terminator
	- Other literals are just values, but C-style literals are **const objects** created at start of program and guaranteed to exist for entirety of program
- Note about char arr and char pointers
	- `const char arr[5] = "Hello";`
	- `const char* p = arr;` //OK. Array decays to pointer
		- Array **decays** ato pointer, but p and arr are fundamentally different types, with different sizes etc.
		- This means p now stores the address of `arr[0]`. That's all
		- eg. `arr` cannot be reassigned, `p` can.

## std::string
- `#include <string>`
- `std::string name { "Alex" };`
- Can use in cout
- Can handle strings of different lengths
	- `name = "bob";`
- `using namespace std::string_literals; // easy access to the s suffix`
	- Then can create string literals with the 's' suffix
	- `"Hello"s`
		- Resolves to `std::string{ "Hello" , 5 }`
- Creates a **copy** of its initialiser
	- Unlike std::stringview, does not have to worry about what happens to the initialise after the initialisation is finished.

## std::stringview (C++17)
- `#include <string>`
- Problem: string copying is slow
	- `std::string s{ "Hello, world!" };`
		- This makes a new copy of the "Hello, world!" literal in s memory
- `std::string_view` provies **read-only** access to existing string
	- `std::string_view s{ "Hello, world!" }; // now a std::string_view`
	- Can pass (s) into a function without copying
- Is a bit like a reference, but 
	- Can bind to many things (`std::string`, string literal, char [])
	- Can be reassigned
	- Can refer to substring
- **NOTE: DOES NOT EXTEND LIFETIME** (unlike references)
	- Important example: 
	- `std::string_view sv = std::string("Hello");  // DANGEROUS`
		- CANT DO THIS!
		- The temporary `std::string` object is **destroyed immediately**
		- Because the `std::string` object is **not a named object**, is just a **temporary object**
	- **So whatever we point sv to has to persistent in memory**
		- eg. can't be getName(), since whatever's returned will be temporary also.
		- It either has to be bound to a name already, or a string literal
- Implementation 
	- Essentially just a `const char *` and `size_t length`.

```CPP

class string_view {
private:
	const char *ptr; //const ptr: what the ptr points to cannot be modified.
	size_t length;

public:
	string_view(char * p, size_t l):p {p}, length{l} {};
	const char& operator[](std::size_t i) { 
		return ptr[i];
	}; // need to return const here since ptr is of type const char*
};

```
- Must explicity cast to std::string if we wwant it
	- `static_cast<std::string>(sv)`
- Can use `sv` suffix to ge a `std::string_view`
- Why not just use `const std::string&`? ie reference to the string?
	- **KIV**
- Modifying a std::string will likely invalidate the std::stringview bound to it
```CPP
std::string s{"Hello"};
std::string_view sv {s}; // sv views s
s = "New"; // s reassigned
// s will return the memory used for the old string data back into os, so that old memory becomes dangling

std::cout <<sv; // undefined behaviour

// however, modifying the memory in place is ok
s[1] = 'x'; //ok
```
- Careful when returning stringview from a function
	- If its bound to a std:: string, that std::string will be destroyed after function returns so sv views a destroyed object
	- But if its bound to a string literal, literals exist throughout program lifetime so ok.
	- Can return if it's the same string_view taken from function parameter
		- Since the objects they view will still exist in the caller's scope
- View modification  and substrings
	- `sv.remove_prefix(1);`
	- `sv.remove_suffix(2);`
	- `sv_sub = sv.substr(7, 5); //index 7, length 5
- Note: may not be null terminated

## When to use what
- Variables
	- std::string
		- Need to modify
		- Need user input text
		- Need store return value of a function that returns this
	- std::string_view
		- Only need read-access to part/all of a string that already exists, and will not be modified/destroyed before use of sv is complete
		- Need symbolic constant for C-style string
		- Need to continue viewing the return value of a function that returns a C-style string/non-dangling std::string_view
- Function param
	- std::string
		- Need to modify string passed in argument without affecting caller (rare)
		- C++14 or older, and dont want to use reference 
	- std::string_view
		- Need read-only string
		- Function needs to work with non null-terminated strings
	- const std::string&
		- Older than C++14, function needs read only string
		- Calling other functions that require a const std::string, const std::string&, or const C-style string (since sd::string_view may not be null-terminated)
	- std::string&
		- Using std::string as out-parameter
		- Calling other functions that require std::string&
- Return types
	- std::string
		- Return value is a std::string local variable or function parameter.
		- Return value is a function call or operator that returns a std::string by value.
	- std::string_view
		- returns a C-style string literal or local std::string_view that has been initialized with a C-style string literal.
		- returns a std::string_view parameter
