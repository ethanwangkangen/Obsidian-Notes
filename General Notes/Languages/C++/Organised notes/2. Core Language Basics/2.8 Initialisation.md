# Brace initialisation
- Specify contents of container
	- `std::vector v{ 1, 3, 5 };`
- Default initialisation values for non static data members
	- `int x{6};`
	- And thus avoiding *most vexing parse*
		- `Widget x();` // this is a function declaration
		- `Widget x{};` // this is variable initialisation

## Deduction under brace initialisation
- Pre C++17
	- ALL of the below deduce to `std::intializer_list<int>`
```cpp
auto x = {1, 2, 3}; 
auto x{1, 2, 3};    
auto x{1};           
```

- Post C++17
	- `=` will result in deduction to `std::intializer_list<int>`
	- Without `=`, if single variable, then deduces as type of that variable, else `error`
```cpp
// copy-list-initialization (equals sign) — still gives initializer_list
auto x = {1, 2, 3};  // std::initializer_list<int>  (unchanged)
auto x = {1};        // std::initializer_list<int>  (unchanged)

// direct-list-initialization (no equals) — NOW deduces the element type
auto x{1};           // int  ✓  (changed!)
auto x{1, 2, 3};     // ERROR in C++17  ✓  (at least it fails loudly now)
```

# Construction with brace initialisation
- If no constructor with `std::initializer_list` parameters, `()` and `{}` have same meaning
	- Else, calls using `{}` **strongly** prefer the overloads taking `std::initializer_lists`
		- eg. `vector<int> {1,2}`
	- Prevails even when the best-match `std::initializer_list` constructor can't be called (ie requires **narrowing** conversions) -> error thrown
	- Only if there's **now way** to convert the arguments to those in `std::initializer_list`, will the normal overload be considered
- Designated initialiser ordering
	- C++ 20: Designated initialisers **must appear in declaration order**
	- eg.
```cpp
struct S {
	int .x;
	int .y;
	int .z;
};
// Does NOT compile in C++20
S s2 = {.z = 3, .y = 2, .x = 1};
```

# Direct inialisation
- You explicitly name the type and provide argumetns
- Compiler considers **all constructors** including explicit ones
```cpp
Foo a(42);       // parens
Foo a{42};       // braces
static_cast<Foo>(42);
```
# Copy initialisation, and `explicit`
- Object is initialisated "from" a value, **without directly invoking a construcor syntactically**
```cpp
Foo a = 42;              // declaration with =
Foo a = {42};            // declaration with = and braces

void take(Foo f);
take(42);                // function argument

Foo give() { return 42; } // return statement

throw Foo(42);           // throw expression

Foo arr[] = {1, 2, 3};   // each array element
```

- The `explicit` keyword
	- Copy initialisation **does not** consider explicit constructors
eg.
```cpp
#include <iostream>

struct Foo {
    Foo() { std::cout << "default\n"; }
    explicit Foo(int x) { std::cout << "int: " << x << "\n"; }
    Foo(double x) { std::cout << "double: " << x << "\n"; }
};

int main() {
    Foo a = 42; // copy initialisation
    Foo b = 3.14; // copy initialisation
    Foo c{42}; // direct initialsiation
}
// Full output
/**
double: 42
double: 3.14
int: 42
**/
```
- First 2 are copy initialisation
	- **Explicit** keyword blocks this!!!

#### Clearing up confusion for self
- Direct vs copy initialisation
	- Is not about braces vs parentheses, is about whether you invoke the constructor directly or not
- Explicit keyword
	- Is not about converting between types, eg. foo(int) called with 3.2, rather has to do with disabling copy initialisation for that constructor
# Empty braces 
- No arguments, **not** empty `std::initializer_list`
- If want to call `std::initializer_list` constructor with empty `std::initializer_list`
	- `Widget w5 ({});`
