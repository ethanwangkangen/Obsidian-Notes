```cpp
class Base {
}

class Derived : public Base {
}
```
- Derived object
	- Derived is a Base
	- Contains a Base part **which is constructed first** and Derived part

# Object slicing
```cpp
Derived d;
Base b = d; // slicing occurs
```
- Only the Base part is copied
- The Derived part is **lost**
- **No slicing for references or pointers**

- Be careful with function parameters!
	- Eg. 
```cpp
void func (const Base base);
Derived d{5};
func(d); // accidental slice!

// To solve this, make func take in a reference instead
void func (const Base& base); // Base reference view into a Derived object.
// Virtual functions will resolve correctly too.
```

### Slicing vector
```cpp
std::vector<Base> v{};
v.push_back(Base{ 5 });    // add a Base object to our vector
v.push_back(Derived{ 6 }); // add a Derived object to our vector
```
Solution
- vector of `Base*`
- vector of reference wrappers

### Franken object problem
```cpp
Derived d1{ 5 };
Derived d2{ 6 };
Base& b{ d2 };

b = d1; // this line is problematic
```
- b points at d2
- Assign d1 to b 
	- b is a Base, and the assignment operator that C++ provides for classes is not virtual
	- So assignment that copies a **Base** is invoked
	- **Only the Base portion of d1 is copied into d2**
- Now d2 has the Base portion of d1 and the Derived portion of d2.

> If Base class is not designed to be instantiated by itself (eg. just an interface class), slicing can be avoided by making the Base class non-copyable



# Pointers
- Upcasting
	- Base pointer can point to Derived (safe), since Derived is a Base
	- Cannot access the members of the Derived portion
- Downcasting
	- Base is not necessarily a Derived, so unsafe unless verified
	- `static_cast<Derived*> (b)` // unsafe
	- `dynamic_cast<Derived*> (b)` // better, runtime check

```cpp
// Upcasting (safe)
Base* d = new Derived();

// Downcasting
Base* b = new Derived();   
Derived* d = static_cast<Derived*>(b); // safe only if b is really derived
Derived* d = dynamic_cast<Derived*>(b); // better, checks actual object type at runtime. Returns nullptr if not safe
```

## Behaviour
- `d.foo()`
	- Say `foo()` is overloaded in Base
	- Will call `Derived::foo()` if not virtual
	- Will call `Base::foo()` if virtual

# References
- Upcasting
	- Base reference to a Derived type
	- Cannot access the members of the Derived portion
- Downcasting
	- `static_cast<Derived*> (b)` // unsafe
	- `dynamic_cast<Derived*> (b)` // better, runtime check
```cpp
// Upcasting
Base& b = someDerived;

// Downcasting
Base& b = someDerived;
Derived& d = static_cast<Derived&>(b);
Derived& d = dynamic_cast<Derived&>(b); // safer
```

## Dynamic casting
- Have pointer to **base class**, but need information that exists only in derived class
	- Downcast: pointer/reference from **base type -> derived type**
```cpp
Base* getDerived() {
	return new Derived{1, "Apple"};
}

Base* b{ getDerived() };
Derived* d{ dynamic_cast<Derived*>(b) }; // use dynamic cast to convert Base pointer into Derived pointer

// Can also do this with references
Derived apple{1, "Apple"};  
Base& b{ apple }; 
Derived& d{ dynamic_cast<Derived&>(b) }; // dynamic cast using a reference instead of a pointer
```
- Dynamic_cast checks that the object is actually what it's casting to!
	- If cast fails, returns **null pointer**
	- Checks that b is polymorphic (**must have at least one virtual function**)
	- Follow vptr to get the object's actual runtime type, then compare
		- Because vtable contains type information!
- Does not work in some scenarios
	- With protected/private inheritance
	- Classes that do not declare or inherit any virtual functions (no vtable)
	- Certain cases involving virtual base classes
```cpp
if (d) ... // make sure dynamic cast succeeds!
```

## Static casting
- Can downcast statically too, but no check!
```cpp
Derived* d{ static_cast<Derived*>(b) }; // must be sure that b is pointing to a Derived object
```

> In general, use static_cast unless downcasting

> In general, avoid dynamic casting and use virtual functions. Unless the derived function doesn't make sense to put in the base class.
