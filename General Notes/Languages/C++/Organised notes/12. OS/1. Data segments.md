```
┌──────────────────────────┐  high addresses
│ kernel space (off-limits)│
├──────────────────────────┤
│ stack          ↓ grows   │  locals, function calls
│                          │
│         (free)           │
│                          │
│ heap           ↑ grows   │  new, malloc
├──────────────────────────┤
│ BSS                      │  uninitialized globals/statics
├──────────────────────────┤
│ data                     │  initialized globals/statics
├──────────────────────────┤
│ text                     │  code, read-only literals
└──────────────────────────┘  low addresses
```

# Text
- Compiled machine code
- Literals
```cpp
char* s = "hello";
s[0] = 'H';  // undefined behavior — likely segfault
```
- Usually read-only and executable
# Data
- **Initialised** static/global data
```cpp
int s = 42;
static int s2 = 1;
```
- Read-write, unless declared const, in which case my go to read-only data section

# BSS
- **Uninitialised** static/global data
- Zero-initialised
```cpp
int g_buffer[1000];        // BSS (zero-initialized by default)
static int s_count;        // BSS
int g_x = 0;               // BSS (explicit zero, but still zero)
```

# Stack
- Local variables, funciton parameters, return addresses, saved registers
# Heap
- Dynamically allocated memory
