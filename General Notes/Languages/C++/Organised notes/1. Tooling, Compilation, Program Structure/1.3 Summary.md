# What goes where
> Essentially, think of headers as a package of forward declarations to copy paste into each .cpp file

- .h files
	- Declarations
		- Class/struct definitions
		- Function declarations
		- Constants
		- Inline functions
		- Templates
- .cpp files
	- Definitions
		- Function implementations
		- Variable definitions
		- Main function

Eg.
- `Vector3d.h` 
```cpp
#pragma once
class Point3d.h; //Forward declaration 

class Vector3d.h {
public:
	Vector3d(double x, double y); // Just declaration only.	

private:
	double m_x{};
	double m_y{};
};
```
- If **functions defined** here, use **inline** to allow multiple definitions (since header might be included in multiple files)
	- `constexpr` function is implicitly inline.

- `Vector3d.cpp`
```CPP
Vector3d::Vector3d(double x, double y): m_x{x}, m_y{y} {
} // Definition of the constructor

//etc.
```

## extern
### for variables
```cpp
// globals.h
#pragma once
extern int counter; //declaration only
```

```cpp
//globals.cppp
#include "globals.h"
int counter = 0; //definition, actual storage
```

```cpp
//main.cpp
#include "globals.h"
void f() { counter++; } //uses the same variable
```

### for functions
- Rarely needed, functions `extern` by default

## inline
- To allow function/variable to be defined in multiple .cpp files (via header) without violating ODR

### for functions
```cpp
// math.h
#pragma once
inline int square(int x) { return x * x; }
```

### for variables
```cpp
// constants.h
#pragma once
inline const int MAX_SIZE = 100;
```

# **Key Decision Rules to Remember**

1. **Functions:**
    - `constexpr` → implicitly inline → safe to put in headers.
    - Normal functions → external linkage by default; `static` for TU-only.

2. **Global Variables:**    
    - `constexpr` → **not inline by default**, must mark `inline` for header-only usage (C++17+).
    - `const` → **internal** linkage by default → safe in header, private to TU. 
	    - Use `extern const` for shared runtime constants.
    - Non-const → default external linkage → avoid if possible; `static` for TU-only.
    
3. **`extern` without initializer** → forward declaration of variable defined elsewhere.

4. **TU-only** → variable or function is visible **only in this translation unit**, safe from linker conflicts. Achieved with `static` or anonymous namespace (for variables/functions).
