# Linkage
- Only applies to **global variables, namespace scope variables and functions**
	- Does not apply for local variables! 
		- These belong to runtime stack frame, exists only while function runs. Invisible to linker.
		- External linkage can only apply for symbols with static storage duration (except for local variables with static, these don't apply still)
- External linkage
	- Symbol can be seen across **multiple** translation units
	- **Non-const global variables** and **functions** have external linkage by default
- Internal linkage
	- Symbol can only be seen across its own tranlsation unit
	- Use **static** to make a **non-const global variable/function** have internal linkage

## extern
- `extern` for **declarations** (linking to a definition elsewhere)
	- No initialiastion
	- `extern int g_x;` // in header file
	- `int g_x = 10;` // in .cpp
	- In this case, `extern` means: this symbol exists, but defined somewhere else
	- Note: functions are `extern` by default, that's why we can forward declare and have the definition somewhere else
- `extern` for **giving external linkage**
	- For **CONST** global variables specifically
	- With initialisation
	- `extern const int g_x = 10;`
		- But this will violate ODR if put in header and included in muiltiple TUs
	- But more commonly, just use `inline`

## inline
- Used to be to request for inline expansion, but this is outdated.
- Inline **functions**
	- Allowed to have multiple (same) definitions, does not break ODR
	- With this, can put function definitions in **headers** safely
	- Compiler must see **full definition** in **each translation unit** where function is used (not just forward declaration)
	- Every definition for inline function must be **identical**
	- **constexpr functions are implicitly inline**
- Inline **variables
	- Similar as above. 
	- Allowed to have multiple (same) definitions, does not break ODR
	- **constexpr variables are NOT implicitly inline (before C++ 17)**

## static
| Context                  | Effect of `static`                                                                                             |
| ------------------------ | -------------------------------------------------------------------------------------------------------------- |
| Global variable/function | Give **internal linkage** to a function/non-const variable                                                     |
| Local variable           | Change lifetime to static rather than automatic (last throughout program). Note that **scope does not change** |
| Class member variable    | Shared across all instances; belongs to class. Eg. `count`                                                     |
| Class member function    | Can be called without an object; cannot access non-static members. eg. `utility_function()`                    |
