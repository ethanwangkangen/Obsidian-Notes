# Ill formed
- Won't compile. The compiler must reject it or t least diagnose it
- It's a straight up language rule violation
- eg. 
	- modifying a const variable


# UB
- Compiles and runs, but standard doesn't say anything about what happens.
- Anything can happen.
- eg.
	- dereferencing a null pointer
	- signed overflow
	- out of bounds
	- double free

# Unspecified
- Standard gives a set of valid options but doesn't say which one the compiler picks
- Compiler doesn't have to tell you or be consistent
- eg. 
	- `int i = f() + g();` // order

# Implementaiton defined
- Compiler does document its choice. Can look it up.
- eg. 
	- `sizeof(int)`
