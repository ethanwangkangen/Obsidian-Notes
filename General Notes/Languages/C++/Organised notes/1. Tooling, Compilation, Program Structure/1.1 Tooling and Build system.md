# Compilation Pipeline
## 1. Preprocessing
- Translation unit 
	- A `.cpp` file **after preprocessing** (`#include`, `#define` have all been expanded)
- `#include <iostream>`, `#include "file.h`
	- Header file code is **copy pasted** into the file
	- Note that `<iostream>` contains `namespace std{...}`, so we have to access those methods with `std::function()`
	- Same header file can be included in multiple files
		- But take note of ODR! See below.
	- `<...>` means that this header was not written by us
	- Header guards
		- To prevent the **same** header from being processed more than once in a **single translation unit**
		- Header guard
			- `#ifndef SOME_UNIQUE_NAME_HERE`
			- `#define SOME_UNIQUE_NAME_HERE`
			- // code
			- `#endif` (at the end of the file)
		- Pragma once
			- `#pragma once`
- `#define`
	- `#define MY_NAME "Alex` replaced with "Alex"
	- `#define USE_YEN` replaced with NOTHING
	- `#define DEBUG`
		- `#ifdef DEBUG`
		- `std::cout << "debug"`
		- `#endig`
## 2. Compilation
- Each `.cpp` file is converted in to a `.o`/`.obj`/`.out` file
## 3. Linking
-  Resolve symbols
	- For each undefined symbol, linker searchers other object files, static libraries (.a, .lib), dynamic/shared libraries (.so, .dll)
- Combine object files into one executable
- Assign final memory addresses for code, global/static variables, constants
