# NULL
- Implemented as `int`, but can also be implemented as other integrals `long`
- Not a pointer type
- **If C++ looks at 0 (int) in context where only a pointer can be used, it will interpret 0 as null pointer, as a fallback position**
	- But note that **there is no actual conversion of int to void****
		- **Once 0 has been captured as an `int` (eg. deduced in templated function), it cannot be converted to `void*`** (see example below)
	- But if it's `0L` or something, then ambiguous since converting `long` to `int` and `long` to `bool` and `0` to `void*` are all equally valid
- Implication:
	- Overloading on pointer and integral types can lead to surprises

# Nullptr
- Type is `std::nullptr_t`
	- Which implicitly converts to **all raw pointer types**
- `f(nullptr)` calls `f(void *)` overload

## Usefulness: templates

```cpp
template<typename FuncType,
 typename MuxType,
 typename PtrType>
decltype(auto) lockAndCall(FuncType func, // C++14
 MuxType& mutex,
 PtrType ptr)
{
 MuxGuard g(mutex);
 return func(ptr);
}
```
- In this example, say call
	- `auto result1 = lockAndCall(f1, f1m, 0)` vs
	- `auto result2 = lockAndCall(f1, f1m, NULL)` vs
	- `auto result2 = lockAndCall(f1, f1m, nullptr)` 
	- Where f1 takes in `void *` as parameter
- The first 2 calls result in **error**!
	- 0 and nullptr passed in
		- `PtrType` deduced as `int` 
		- Cannot pass into call that takes in `void *`
