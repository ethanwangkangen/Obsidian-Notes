# Normal heap allocation
## normal new
- This does **two** things
	- `int *p = new T(4);`
- 1. **Allocates memory** large enough to store the object
- 2. **Constructs** the object at that memory, calling the relevant constructor
- Finally, returns the address of that memory

## normal delete
- `delete p`
- 1. **Calls the destructor** on that object
- 2. **frees** the memory there

- The same applies for `new []` and `delete []`. The objects are constructed

# Placement new
- `new (address) int(4);`
- Does **NOT** allocate any memory
- Takes in memory that **already has been allocated**, then constructs the object there
- Then returns the address of that memory.

### There is NO placement delete
- Must call the destructor manually.
```cpp
p->~T();
```

## Applications of placement new
- Custom allocators/ memory pools
```cpp
alignas(Order) char pool[sizeof(Order) * 1024];
int idx = 0;

Order* o = new (pool + idx * sizeof(Order)) Order{...};
idx++;
```

- How `std::vector` works internally

- Reinitialising without allocating
```cpp
p->~Widget();
new (p) Widget(new_args);
```

- Aligned/shared/mapped memory
	- When memory comes from mmap, shared memory segments or hardware-mapped regions, can't use normal `new`
