# Dynamic allocation
- Memory allocated on the heap
- No automatic freeing, must free manually
```cpp
new int; // allocate integer, returns a pointer containing address of memory allocated

int* ptr{new int}; // store pointer in a variable
*ptr = 7; // dereference pointer to access the memory

// Deleting
delete ptr; // return memory to OS
ptr = nullptr;
```

# Failed allocation
Failed allocation
- Throws bad_alloc exception
- Can use std::nothrow to return a null pointer instead

```cpp
int* value { new (std::nothrow) int{} }; 
if (!value){
    // error handling here
    std::cerr << "Could not allocate memory\n";
}
```

# Memory leaks
- If ptr not freed, but goes out of scope
	- ie. function returns before ptr is freed
	- No way to free the memory allocated anymore!
- If ptr is assigned another value before old memory is freed.
```cpp
void doSomething() {
    int* ptr{ new int{} }; // not freed, goes out of scope
}

int value {6};
int* ptr{ new int{} }; // allocate memory
// delete ptr; //should do this first. return memory
ptr = &value; // old address lost, memory leak results
```

## Dynamically allocating C-style arrays
- Note: can dynamically allocate `std::array`, but then should probably use `std::vector`
- Use the array forms, `new[]`, `delete[]`

```cpp
std::cout << "Enter a positive integer: ";
std::size_t length{};
std::cin >> length;


int* array{ new int[length]{} }; // use array new.  Note that length does not need to be constant!
// array is int pointer to first element. just like what a statically allocated array variable decays to!
array[0] = 5;
delete[] array; // use array delete to deallocate array
```
