```cpp
std::allocator<int> alloc;

// allocate — raw memory for n objects, no constructors called
int* p = alloc.allocate(5);     // space for 5 ints

// deallocate — free that raw memory, no destructors called
alloc.deallocate(p, 5);         // must pass same n
```
- Does **NOT** construct any object at the memory
- Must be done separately
	- Consequently, must **delete** those objects manually as well
	- (call the destructor of those objects, in case the destructors do special things)

# How to construct/delete objects there?
- `std::constuct_at()` and `std::destroy_at()`
```cpp
int* p = alloc.allocate(3);

// construct objects in that memory
std::construct_at(p,     10);   // p[0] = 10
std::construct_at(p + 1, 20);   // p[1] = 20

// destroy objects (calls destructors)
std::destroy_at(p);
std::destroy_at(p + 1);
```
