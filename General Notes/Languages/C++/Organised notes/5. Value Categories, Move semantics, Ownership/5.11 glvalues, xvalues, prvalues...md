```
        expression
       /          \
    glvalue      rvalue
    /    \       /    \
lvalue     xvalue    prvalue
```
- **glvalue** — "**g**eneralized lvalue." Anything with identity (lvalue or xvalue).
- **rvalue** — Anything you can move from (xvalue or prvalue).
- **lvalue** — "**l**ocatable." Has a name or address, sticks around. `int x; x` is an lvalue.
- **prvalue** — "**p**ure rvalue." A temporary with no identity. `42`, `x + y`, `std::string("hi")`.
- **xvalue** — "e**x**piring value." Has identity _but_ you've explicitly said "go ahead, move from me." The result of `std::move(x)` or `static_cast<T&&>(x)`.

**Quick test you can apply to any expression `e`:**
1. Can you write `&e`? → it has identity (glvalue)
2. Can it bind to `T&&`? → it's movable (rvalue)
3. Both → **xvalue**. Only 1 → **lvalue**. Only 2 → **prvalue**.

```cpp
int x = 10;
int* p = &x;
std::string s = "hey";
std::string foo() { return "hi"; }

// lvalue — has address, sticks around
x
*p
s[0]
"hello"    // surprise: string literals are lvalues
++x

// prvalue — temporary, no address
42
x + 1
foo()
std::string("tmp")
x++        // returns a copy, not x itself

// xvalue — has identity but marked "take my guts"
std::move(s)
static_cast<std::string&&>(s)

// glvalue = lvalue + xvalue
// rvalue  = xvalue + prvalue

```
