- `decltype(auto)` **preserves references and cv-qualifiers**
```cpp
int i = 5;
int& r = i;

decltype(auto) x = r;   // x is int& (reference preserved)
decltype(auto) y = i;   // y is int (i is a named lvalue, but decltype(i) is int)
decltype(auto) z = (i); // z is int& — parens matter! (i) is an lvalue expression
```

# Use case - forwarding return types
- Just like how `std::forward<T>` was used to preserve value of arguments, `decltype(auto)` is used to forward a return value with its exact type

```cpp
template <typename F, typename... Args>
decltype(auto) call(F&& f, Args&&... args) {
    return std::forward<F>(f)(std::forward<Args>(args)...);
}
```
- Return type is `decltype(auto)`
	- If it was just `auto`, references would be stripped.
	- Now if f returns `int&`, call returns `int&`
		- If it returns `int&&`, call returns `int&&`
		- If it returns `int`, call returns `int`

# Watch out for ()
```cpp
decltype(auto) f() {
    int x = 5;
    return x;     // returns int
}

decltype(auto) g() {
    int x = 5;
    return (x);   // returns int& — DANGLING REFERENCE to local
}
```
- `(x)` becomes an **lvalue expression** so `decltype(auto)` deduces to `int&`
