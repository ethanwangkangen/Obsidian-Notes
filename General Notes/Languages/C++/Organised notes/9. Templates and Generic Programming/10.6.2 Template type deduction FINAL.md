> Deduce T first, then apply T to the parameter form (with collapsing) to get parameter type

- First
	- **IGNORE** the **referenced**-ness of the **input**. References on the argument are always transparent
	- What matters is the value category (rvalue or lvalue) and underlying type (including const-ness)
- Then look at how the template is
	- T
		- Always clean type. Ignore constness of input.
		- Decay converts arrays to pointers and functions to function poitners. This doesnt happen for reference templates.
	- T&
		- Absorbs whatever the argument has, because the parameter needs to know exactly what its referring to.
		- If argument is const, T must be const too.
	- const T&
		- Parameter already promises const, so T doesnt need to carry it
	- T&&
		- First check input
			- Input is lvalue: T is U&
			- Input is rvalue: T is U
			- Constness is kept as well.
		- Then from T, get the parameter using **colapsing rules**
		- Lvalue makes T a reference (so collapsing produces lvalue ref). Rvalue keeps T clean (so you get rvalue ref). This is the entire mechanism that makes `std::forward` work.
	- const T&&
		- TRAP!!
		- This is **no longer a forwarding reference**
		- Only binds rvalues, and const is absorbed

One trick for remembering: if the parameter **already has const**, T doesn't need it. If it **doesn't**, T keeps whatever the argument had.
