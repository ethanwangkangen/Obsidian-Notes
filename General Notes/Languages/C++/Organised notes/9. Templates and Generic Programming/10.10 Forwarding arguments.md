```cpp
template <typename T, typename... Args>
void doSomeWork(Args&&... params) {
	T localObject(std::forward<Args>(params)...);
}
```

# Std::forward
- Preserves the **value category (lvalue vs rvalue)** of argument when passing it through **function template**
- When have a template function taking a universal reference T&&, the parameter itself is **always an lvalue**, even if it was bound to an rvalue
	- Since it is named

## Mechanism
- Relies on reference collapsing
- If T is deduced as U& (lvalue passed in), T&& collapses to U&, and `forward<T>` returns an lvalue reference.
- If T is deduced as U (rvalue passed in), T&& stays U&&, and `forward<T>` returns an rvalue reference (enabling moves).

```cpp
// How std::forward is implemented under the hoot conceptually
template <typename T>
T&& forward(std::remove_reference_t<T>& arg) noexcept {
    return static_cast<T&&>(arg);
}
```

# Use cases

## std::make_unique
- `std::make_unique<T>`
```cpp
template<typename T, typename... Args>
std::unique_ptr<T> std::make_unique<T> (Args&&... args){
	return unique_ptr<T>(new T(std::forward<Args>(args)...));
}
```
- Whatever passed to `make_unique` is **forwarded** to T's consturctor with value category intact
	- So moveable temporary stays moveable and lvalue stays lvalue

## Templated forwarding constructor
- What if my constructor has multiple parameters, and each can take in either lvalue or rvalue? 
	- We don't want to have so many different overloads
```cpp
struct T{
	template<typename AA, typename BB>
	T (A&& a_, B&& b_) : a(std::Forward<AA> (a)), b(a(std::Forward<BB> (B))) {
	}
};
```
- A&& and B&& are now **forwarding references/universal references**
- The parameter (A&&) will be deduced to the correct type (lvalue or rvalue reference)
	- See [[10.6 Template type detection]]
