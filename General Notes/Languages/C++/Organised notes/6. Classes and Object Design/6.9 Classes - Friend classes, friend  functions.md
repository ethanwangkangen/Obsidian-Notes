# Friend classes
- Compiler must see whole definition!
	- Forward declaration satisfies **reference** to the **class**, but NOT references to the **class members**
	- To access the members, needs to see **whole definition**
- (See example below)
	- Storage declares Display as a friend
	- Display can now acces Storage's **private variables**
```cpp
class Storage {
    // Make the Display class a friend of Storage
    friend class Display;
};
```

# Friend functions
- Ok to have forward declaration only.

```cpp
class Display {
private:
	int x;
	  
	friend void Storage::f();  // member function f is defined in Storage
	friend void g(); // non-member function
};

class Storage {
	void f(Display& d) {
		std::cout << d.x;
	}
}
```
- `Storage::f()` and `g()` now can access private members of Display



# Notes for actual code, and compiling in .cpp and .h files
- Prefer forward declarations to inclusion
	- Especially for friends, eg. Vector3d has a member function that is declared friend of Point3d. Cannot include Vector.h in Point3d.h and vice versa, cyclic dependency!
		- Instead, **forward declare** Vector3d in Point3d.h
	- Include if 
		- Must use object directly
		- Inherit
		- Use infline functions involving the type
- Remember to end off class defintiions with `;`
