# Non-const function that uses the const version
- Prevent deduplication of code between const and non-const functions
- Casts the object into a **const reference**
	- `static_cast<const Buffer&>(*this)`
- Then, invoking the function will call the pre-existing **const** version
```cpp
class Buffer {
    char* data_;
    size_t size_;
public:
    const char& operator[](size_t idx) const {
        // imagine bounds checking, logging, complex logic here
        return data_[idx];
    }

    // Without the idiom: copy-paste the entire body, just non-const.
    // With the idiom: delegate to the const version.
    char& operator[](size_t idx) {
        // 1. cast *this to const to call the const overload
        // 2. const_cast away the const from the result
        return const_cast<char&>(
            static_cast<const Buffer&>(*this)[idx]
        );
    }
};
```

# C++23 version
- Uses **explicit object parameters** to do this
- Self deduces as `Buffer&` or `const Buffer&` depending on the object's constness
```cpp
cppclass Buffer {
public:
    // Single function handles both const and non-const
    template <typename Self>
    auto&& operator[](this Self&& self, size_t idx) {
        return std::forward<Self>(self).data_[idx];
    }
};
```
