# Difference between Structs and Classes
- Default **member** access
	- Struct - **public** by default
	- Class - **private** by default
- Default **inheritance** access
	- Struct - **public** by default
	- Class - **private** by default

Those 2 are the **only** differences!
- Classes can be aggregates too

But generally we think of structs as aggregate
- **Avoid defining constructor, member functions as doing so makes them non-aggregate**

> Use struct when the type would be an **aggregate**, or is purely data with **no invariants** to maintain
> Use class when the type has **private** data and needs to **enforce invariants** through its interface

**For the purposes of these notes, assume that the following sections have struct that is aggregate.**

# Initialisation
- Aggregate initialisation with initialiser list
- `joe {1, 33, 44000.0}; // in order of member declaration`

# Assignment
- Can do with initialiser list too
	- `joe = { joe.id, 33, 66000.0 }; //update values all at once`
- This is copy assignment with a temporary aggregate on the right hand side.

# Passing structs
- Usually by **const ref**
	- `void function(const Employee& employee)`
- Can pass in temporary objects
	- `printEmployee(Employee { 14, 32, 24.15 });` explicit typing
	- `printEmployee({14, 32, 24.15})` type deduced from parameter

# Returning structs
- Usually return by value
- Type return deduction
	- SImilar to parameter type deduction (above)
	- `return {0, 1, 2};`

# Const and constexpr structs
- Must be initialised
```
const Rectangle unit { 1.0, 1.0 }
```
- Const struct
	- Promise that object won't be modified
- Constexpr struct
	- Object is **evaluated** at **compile time**
	- Struct and its initliasation must be **fully evaluable at compile time**
		- No runtime values, no dynamic allocation etc.

# Structs within program-defined members
```CPP
struct Company {
	Employee CEO{};
}
```

# Nested structs
- Can nest enum within struct too. Eg. enum Type{...} within struct Monster{..}
```cpp
struct Company {
    struct Employee { // accessed via Company::Employee
        int id{};
        int age{};
        double wage{};
    };
    int numberOfEmployees{};
    Employee CEO{}; // Employee is a struct within the Company struct
};
```

# Struct size and data alignment
- At least as large as size of all variables it contains
	- But could be large!
		- Padding
	- Reduce this by defining members in decreasing order of size
	- Keep in mind, C++ does not reorder members.

# Member selection from pointer operator (->)
- `->` is the same as `(*).`
```CPP
Employee joe {1,2,3};
Employee* ptr {&joe};

(*ptr).age +=1;
// is equivalent to
ptr->age +=1;
