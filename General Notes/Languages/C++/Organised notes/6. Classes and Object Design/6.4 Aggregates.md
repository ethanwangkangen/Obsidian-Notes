# Types
- C-style Arrays
- Class types (struct, class, union) that meet the following conditions

# Conditions for class types
- No user-provided constructors
- No private or protected non-static data members
- No virtual functions
- No virtual, private, or protected base classes

# Key properties
- Members are intiialised in declaration order
- Omitted members are zero-initialised (or use their default values)
- No encapsulation
	- Aggregates are just plain data holders

```cpp
struct Point {
    int x;
    int y;
}; // aggregate

Point p{ 1, 2 }; // aggregate initialization, x=1, y=2
```

# Aggregate initialisation
```cpp
Point p{ 1, 2 }; // aggregate initialization, x=1, y=2
int arr[5]{ 1, 2, 3 }; // arr = {1, 2, 3, 0, 0}
```
- **Initialiser list** as initialiser
- **Memberwise** initialisation
	- Each member initialised in order of declaration
- If have missing initalisers
	- if member has **default member initialiser**, then that is used
	- If not, member is **copy initialised** from **empty intialiser list** (ie. zero initialised)
	- This means we can use empty initialisation list to value initialise all members of struct
		- `Employee joe {};`

- Note that both `Employee joe;` and `Employee joe {};` allowed, but only the latter will value initialise all members, **even in the case that default values not given** .
	- ie. `int age;` -> missing default values.
	- `Employee joe;` -> age is not value initialised, so undefined
	- `Employee joe{}` -> age is copy initialised from empty intialiser list, ie 0.
