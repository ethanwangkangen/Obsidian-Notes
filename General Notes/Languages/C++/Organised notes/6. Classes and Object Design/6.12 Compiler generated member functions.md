# Rules for the 6 special member functions
- Default constructor
	- Generated only if **no constructor** declared by user
- Copy constructor,  copy assignment operator
	- Generated only if **no move members** declared by user (but deprecated)
- Move constructor, move assignment
	- Generated if **none of the others** (*except constructor*) declared by user

Note:
- `Cow& operator= (const Cow&) = delete;`
	- **delete** also counts as declared by users!

## What function signatures count?
**Copy constructor**
- any constructor whose first parameter is `T&`, `const T&`, `volatile T&`, or `const volatile T&`, with all remaining parameters having defaults

```cpp
T(const T&);            // the usual one
T(T&);                  // non-const, still a copy ctor
T(const T&, int x = 5); // extra defaulted param, still a copy ctor
```

**Copy assignment**
- any `operator=` taking `T`, `T&`, `const T&`, `volatile T&`, or `const volatile`
- taking by value counts:

```cpp
T& operator=(const T&);  // usual
T& operator=(T);          // by value — still copy assignment
```

**Move constructor**
- first parameter is `T&&`, remaining parameters have defaults:

```cpp
T(T&&);
T(T&&, int x = 0);  // still a move ctor
```

**Move assignment**  `operator=` taking `T&&`.
```cpp
T& operator(T&&);
```

- The **compiler-generated** versions always use the "usual" signatures (`const T&` for copy, `T&&` for move). But when the standard says "if you declare a copy constructor, moves are suppressed" — it means if you declare _any_ of these broader forms, not just the exact signature the compiler would generate.

# Rules of 3, 5, 0
**Rule of Three** (pre-C++11): if you define any of {destructor, copy ctor, copy assign}, you should define all three, because if one needs custom logic, the others probably do too.

**Rule of Five** (C++11+): extends to all five: {destructor, copy ctor, copy assign, move ctor, move assign}.

**Rule of Zero** (modern best practice): don't declare _any_ of them. Use RAII types (`unique_ptr`, `vector`, etc.) as members so the compiler-generated versions are correct by default.
