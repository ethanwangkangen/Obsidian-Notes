# Scoped Enum (Enum classes)
```cpp
enum class Color{ //keyword 'class'
    red, // red is considered part of Color's scope region
    blue,
};

std::cout << Color::red;
```

- **No implicit conversion to int**, but can explicitly convert
	- This is the **main point** of scoped enum, no accidentally implicit conversion toi nt

```
std::cout << static_cast<int>(color) << '\n'; 
```
- More advanced way, overload the `operator+` to convert to int.
```CPP
template <typename T>
constexpr auto operator+(T a) noexcept
{
    return static_cast<std::underlying_type_t<T>>(a);
}
```

- Scope is contained within the class

- Can cast integer to scoped enumerator, useful when taking in input
```CPP
Pet pet{ static_cast<Pet>(input) }; // static_cast our integer to a Pet
```

# Underlying type
- Default int, but can change
	- Same as unscoped enum
```cpp
enum class A : int8_t { X = 127 }; // -128 to 127
```
## Values not explicitly defined
```cpp
enum class FeePriority {
    One = 0,
    Two,    // 1
    Three   // 2
};
FeePriority a = FeePriority(3);  // ✅ Legal: 3 is range of int
```
- Can hold values outside the explicitly defined ones
- The range is defined by the **underlying type (int default)**
	- **Different from unscoped enum!**

# Use enum class over enum (scoped over unscoped)
- No implicit conversions to any other type
- Ability to forward declare without specifying enumerators 
	- `enum class Color;` // ok
	- `enum Color;` // not ok
	- Why better -> if `Color` definition is changed, don't have to recompile everything 
	- This is possible because C++ knows the underlying type of a scoped enum (int by default)
