# Why?
- Every blocking call seen before (`join`, `cv.wait`, `future.get`, mutex lock) blocks **indefinitely**
	- Sometimes we want a bound
- 2 kinds of timeout
	- Duration based
		- `_for` suffix
		- Which takes in a `duration`
		- Uses internal steady clock which is immune to changes in system clock etc.
	- Absolute
		- `_until` suffix
		- Takes in `time_point`
			- Clock parameter affects this! 

# Clock
- A clock class has 4 things
	- Time now 
		- via static `some_clock::now()`,
		- Returns a `some_clock::time_point`
	- Type used to represent times
		- The `time_point` member typedef
	- Tick period
		- `period` member typedef
		- `std::ratio` of seconds-per-tick
		- one tick per 2.5s -> `ratio<5,2>`
	- Whether it's steady
		- Static `is_steady` data member
		- steady meaning: clock ticks at uniform rate and **can't be adjusted**
			- Matters because a non-steady clock can be adjusted such that `now()` returns a value earlier than a prior call

3 standard clocks
- `std::chrono::system_clock`
	- Typically not steady
- `std::chrono::steady_clock`
	- Guaranteed steady
- `std::chrono::high_resolution_clock`
	- Smallest possible tick period
	- May be typedef for one of the other 2 - don't assume steady

# Duration
- `std::chrono:duration<Rep, Period>`
	- Rep: representation (int, long, double)
	- Period: `std::ratio`, seconds-per-unit
	- eg.
		- `duration<short, ratio<60,1>>` - minutes in a short
- Predefined typedefs
	- `nanoseconds`, `microseconds`, `milliseconds`, `seconds`, `minutes`, `hours`
- Predefined literal suffix operators
	- `using namespace std::chronoe_literals`
		- eg. `24h`, `15ns`, `30min`
- Conversion between durations
	- Implicit where it does not require truncation of value (eg. converting hours to seconds ok, but not seconds to hours)
	- Explicit conversions can be done with `std::chrono::duration_cast<>`
		- Value is truncated
```cpp
std::chrono::milliseconds ms (54802);
std::chrono::seconds s = std::chrono::duration_cast<std::chrono::seconds> (ms);
```
- Arithmetic
	- Add/subtract durations
	- Multiply/divide by a `Rep` constant
	- `.count()` returns the unit count
## Example duration
```cpp
std::future<int> f = std::async(some_task);
if (f.wait_for(std::chrono::milliseconds(35)) == std::future_status::ready)
    do_something_with(f.get());
```
- Wait functions return a **status**
	- For futures - `future_status::ready`/`::timeout`/`::deferred`
- Key guarantee
	- A `_for` wait is measured by libary-internal steady clock so 35ms means 35ms elapsed time even if system clock is adjusted mid wait

# Time points
- `std::chrono::time_point<Clock, Duration>`
	- a length of time (in units of Duration) since the clock's epoch.
	- Epoch -> clock property you cannot query directly
	- `time_since_epoch()` returns duration from epoch to this point
- Operations
	- time_point += duration = time_point
		- eg. `high_resolution_clock::now() + nanoseconds(500)` -> 500ns in future
	- time_point - time_point = duration
		- Calculating duration elapsed
eg.
```cpp
auto start = std::chrono::high_resolution_clock::now();
do_something();
auto stop = std::chrono::high_resolution_clock::now();
std::cout << std::chrono::duration<double>(stop - start).count() << " seconds";
```

# cv with timeout
- Want to wait for a certain amount of time, then **give up**
- Use `wait_until` instead of `wait_for`
	- If use `wait_for` -> re-entering the loop upon **spurious wakeup** (random, unprompted wakeup) resets the timing, hence unbounded time loop results.
```cpp
bool wait_loop() {
    auto const timeout = std::chrono::steady_clock::now()
                       + std::chrono::milliseconds(500);
    std::unique_lock<std::mutex> lk(m);
    while (!done) {
        if (cv.wait_until(lk, timeout) == std::cv_status::timeout)
            break;
    }
    return done;
}
```

# Functions

| Class / Namespace                                                                       | `_for` (duration)                     | `_until` (time_point)                     | Return value                                                                        |
| --------------------------------------------------------------------------------------- | ------------------------------------- | ----------------------------------------- | ----------------------------------------------------------------------------------- |
| **`std::this_thread`**                                                                  | `sleep_for(duration)`                 | `sleep_until(time_point)`                 | N/A (void)                                                                          |
| **`std::condition_variable`** / **`std::condition_variable_any`**                       | `wait_for(lock, duration)`            | `wait_until(lock, time_point)`            | `std::cv_status::timeout` or `std::cv_status::no_timeout`                           |
| same, predicate overloads                                                               | `wait_for(lock, duration, predicate)` | `wait_until(lock, time_point, predicate)` | `bool` — the value of the predicate when woken                                      |
| **`std::timed_mutex`**, **`std::recursive_timed_mutex`**, **`std::shared_timed_mutex`** | `try_lock_for(duration)`              | `try_lock_until(time_point)`              | `bool` — true if the lock was acquired, false otherwise                             |
| **`std::shared_timed_mutex`**                                                           | `try_lock_shared_for(duration)`       | `try_lock_shared_until(time_point)`       | `bool` — true if the lock was acquired, false otherwise                             |
| **`std::unique_lock`** (constructor)                                                    | `unique_lock(lockable, duration)`     | `unique_lock(lockable, time_point)`       | N/A — `owns_lock()` on the constructed object returns true if the lock was acquired |
| **`std::unique_lock`** (member)                                                         | `try_lock_for(duration)`              | `try_lock_until(time_point)`              | `bool` — true if the lock was acquired, false otherwise                             |

| **`std::shared_lock`** (member)                                                         | `try_lock_for(duration)`              | `try_lock_until(time_point)`              | `bool` — true if the lock was acquired, false otherwise                             |
| **`std::future`** / **`std::shared_future`**                                            | `wait_for(duration)`                  | `wait_until(time_point)`                  |                                                                                     |
