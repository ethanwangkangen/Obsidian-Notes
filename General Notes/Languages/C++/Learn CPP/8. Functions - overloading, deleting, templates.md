# Function overloading
Function signature
- Parts of a function header that are used for differentiation of the function
- Name, number of parameters, parameter types, function level qualifiers
	- Does **not** include return type

- Same name, different parameter list
	- **return type does not matter**. Cannot differ by return type only
	- `const` on parameters does not count for overloading.
	- `const` on member types **does** count
```cpp
struct S {
    void f() const;  // const member
    void f();        // non-const member
};

S s;
s.f(); // calls non-const version
const S cs;
cs.f(); // calls const version
```
- reference types matter
	- `int&` binds only to lvalues.
	- `const int&` can bind to rvalues.
```cpp
void g(int& x);
void g(const int& x);

int a = 5;
g(a);   // calls g(int&)
g(10);  // calls g(const int&)
```

- overloading with pointers/derived classes
	- overload resolution prefers most derived/exact match
	- implicit conversions used only if no exact match exists
```cpp
struct Base {};
struct Derived : Base {};

void h(Base* b);
void h(Derived* d);

Derived d;
h(&d); // calls h(Derived*), exact match preferred

Overload resolution prefers most derived / exact match.
```

- Rules (in order of hierarchy)
	- Exact match: argument types match paramter types exactly
	- Promotion: smaller integer types promoted (char->int)
	- Standard conversion: numeric converison, pointer conversions, derived->base pointer..
	- User-defined conversion calls a constructor or conversion operator
	- Ellipsis ... matches anything, lowest priority
- Note: if multiple matches in one step, ambiguous function call will result.
	- Note: int to unsigned int also considered a conversion. So
	```cpp
	foo(unsigned int) {..}
	foo(float) {..}
	foo(3.145); //ambiguous! can convert to either one
	foo(0); //ambiguous! can convert to either one

	```
- Default arguments can complicate things
```
void f(int x);
void f(int x, int y = 0);

f(5); // ❌ Ambiguous: could call either
```

- When multiple arguments
	- Rules applied to each argument
	- Function chosen is the one for which each argument matches at least as well as all other function,s with at least one matching better than all the others
		- Better as in less steps needed

	- "I forbid this"

# Default arguments
- `void foo(int y, int x = 5);`
- Use case: if may or may not have parameter to pass in.
```cpp
void printIDNumber(const int *id=nullptr) {
    if (id)
        std::cout << "Your ID number is " << *id << ".\n";
    else
        std::cout << "Your ID number is not known.\n";
}

printIDNumber(); // we don't know the user's ID yet

int userid { 34 };
printIDNumber(&userid); // we know the user's ID now
```

- Copy initialisation only
- Explicit arguments must be to the left
- Cannot be redeclared
	- Either in forward declaration and function declaration, **not both**
- Must be declared before use
- Not part of function's signature, so can be differentiated overloads
- Don't work for functions through function pointers

> Best practice: **declare** default argument in **forward declaration** instead of functioon definition, since forward declaration is more likely to be seen by other files and included before use (espsecially if it's in header file)

# Templates
- Compile-time functinon, blueprint for generating code
- Can generate
	- Classes
	- Functions
	- Variables

3 types of template parameters
- **Type** template parameters 
	- template parameter represents a **type**
```cpp
template <typename T> //function generated
T add(T a, T b) {
    return a + b;
}

add<int> (a,b);
```
- **Non-type** template parameters 
	- template parameter represents a **constexpr** value
```cpp
template <int N> // object generated
class Array {
public:
    int data[N];
};
Array<100> b;


template <int N>
void printNTimes() {
    for (int i = 0; i < N; i++)
        std::cout << "Hello\n";
}
printNTimes<3>();
```
- **Template** template parameters
	- template parameter represents a **template**
```cpp
template <template <typename> class Container> //object generated
class Wrapper {
    Container<int> data;
};
Wrapper<std::vector> w;
```

### Type parameters
`template <typename T>`
What happens
- Function instantiation (the first time a function call is made)
	- Replace T with the specified type

Argument deduction
- 2 ways to do it
	- `add<>(a,b);`
		- Templates only.
	- `add(a,b);`
		- Both non-template and template considered, **non-template preferred**
			- (but only prefer non template under equal footing). Normal rules for precedence still apply first.
		- Prefer using this

- Can use default arguments for non-template parameters
- Note: if template function has a static variable, differently instantiated functions do **not** share the same static variable.
- Multiple template types
	- `T max(T x, T y)`
	- `max(1.5,3)`
		- Won't compile!
		- Because T can only represent a single type
		- Why compiler not generate `max<double>` then convert int to touble?
			- Because type conversion is done only when resolving **function overloads**, NOT when performing type deductions
	- Solutions
		- Static convert to same type
			- `max(static_cast<double>(2), 3.5)
		- Don't use type deduction
			- `max<double>(2, 3.5)`
			- Type convert will work here
		- Use different parameter types
			- `auto max (T x, U y)`
			- To forward declare this, use **std::common_type_t**
				- `auto max(T x, U y) -> std::common_type_t<T, U>;`
- Abbreviated function templates
	- C++20
	- (return to this)
- Can overload function templates
	- `auto add (T x, T y)`
	- `auto add (Tx, U y)` //overloaded
		- Note: more restrictive function template chosen. eg. the above former is more restrictive, so preferered, if both possible

### Non-type parameters
`template <int N>`
`template <bool B>` etc.

- Usually, for passing constexpr values to functions (or class types) so they can be used in contexts that require constant expression
	- **Function parameters cannot be constexpr**
	- So this is the solution
eg.
```CPP
double getSqrt(double d)
{
    assert(d >= 0.0 && "getSqrt(): d must be non-negative");

    // The assert above will probably be compiled out in non-debug builds
    if (d >= 0)
        return std::sqrt(d);

    return 0.0;
}
// This works, but we cannot do static_assert() since the d is not constexpr (function param can't be constexpr)


template <double D> // requires C++20 for floating point non-type parameters
double getSqrt()
{
    static_assert(D >= 0.0, "getSqrt(): D must be non-negative");

    if constexpr (D >= 0) // ignore the constexpr here for this example
        return std::sqrt(D); // strangely, std::sqrt isn't a constexpr function (until C++26)

    return 0.0;
}
```

## Templates in multiple files
- Cannot define in add.cpp and forward declare + use in main.cpp file.
	- Because main.cpp doesn't see definition, can't instantiate the function
- Solution: put **all template code in header file**
	- Does not violate ODR, since ODR says that types, templates, inline functions and inline variables can have identical definitions in different files
