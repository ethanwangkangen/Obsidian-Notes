Note that the evaluation of operands (including function arguments) is mostly unspecified
- eg. 
- `printCalculation(getValue(), getValue(), getValue());`
	- Unspecified which getValue() will be called first.
- Clang compiler evaluates arguments in left to right order, GCC compiler evaluates arguments in right to left order

Increment/decrement operators
- ++x: increment x then return x
- x++: copy x, then increment x, then return the increment.
	- returns an rvalue! temporary.
	- `y = x++`
	- original x is unchanged.
	- y = x+1
- Remember that function argument evaluation order is undefined in C++
	- so something like `add(x, ++x)`is undefined.
	- Don't use a variable that has a side effect applied to it more than once in given statement

Comma
- x,y
	- Evaluate x, then evaluate y. Return y
- Try to avoid this

Conditional
- c ? x : y
	- If c is true then evaluate x, otherwise evaluate y
- `int x { bool1 ? 3 : 4}`
- Sometimes need to use.
	- Might try replacing with **if-else**, but variables defined in an if-statement **die at the end of the if-statement**
- Advise to use parentheses since condition is low priority
- Type of expressions must match or be convertible

Comparisons
- Avoid == when dealing with floating point operands
	- Due to rounding errors
- Also not safe to compare floating point literals with different types
	- eg. `9.8f == 9.8` false
