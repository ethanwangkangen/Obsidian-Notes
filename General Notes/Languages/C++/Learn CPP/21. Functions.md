# Function pointers
```cpp
int foo() {
	return 0;
}

int main() {
	std:: cout << foo;
}
```
- The above prints '1', since `foo` is function pointer, and `operator<<` doesn't know how to print that and converts to `bool`.

# Pointers to functions
`int (*fcnPtr)();`
- Non-const function pointer
	- Pointer to a function that returns an int
Reading syntax like this:
https://c-faq.com/decl/spiral.anderson.html
https://web.archive.org/web/20110818081319/http://ieng9.ucsd.edu/~cs30x/rt_lt.rule.html

- C++ will implicitly convert function into function pointer if needed
	- But will not convert to void poitners or vice versa
```cpp
fcnPtr = &goo; // ok
fcnPtr = goo; // also ok! implicitly converted. 
```

- Calling 
	- `(*fcnPtr)(6);` // dereference
	- `fcnPtr(6);` // implicit dereference

- Default argument won't work for functions called through function poitners
	- Resolution happens at runtime. Default arguments not resolved.
	- For normal functions with default arguments, compiler **rewrites** function call to include the defaul targument.

- Can set default parameter, for function that takes in function
	- `void selectionSort(int* array, int size, bool (*comparisonFcn)(int, int) = ascending);`

- Making names nicer: use `using`
	- `bool (*comparisonFcn)(int, int)`
		- Drop the name
		- Type becomes `bool (*) (int, int)`
		- `using ValidateFunction = bool(*)(int, int);`
		- `ValidateFunction comparisonFcn;`
