# Class template

```cpp
template <typename T>
struct Pair {
    T first{};
    T second{};
};
Pair<int> p1{ 5, 6 };
```

## Class template in function
Using class template in a function
```cpp
template <typename T>
constexpr T max(Pair<T> p) {
    return (p.first < p.second ? p.second : p.first);
}
```

Can write function templates that we want to use with any type at all
```cpp
template <typename T>
void print(T p){ // type template parameter will match anything
    std::cout << '[' << p.first << ", " << p.second << ']'; // will only compile if type has first and second members
}

```

> Stick to T, U, N etc. If not, may shadow class name, and end up referring to template when wanting to specify a specific class type

## Class templates in multiple files
- Like function templates, typically defined in header files
	- Template definitions and type definitions exempt from ODR

## Class Template Argument Deduction (CTAD)
- Unlike function template argument detection,
	- Cannot do `pair<>`
	- Don't put `<>` at all
```cpp
std::pair<int, int> p1{ 1, 2 }; // standard, explicit specifying
std::pair p2{ 1, 2 }; // CTAD used.

std::pair<> p1 { 1, 2 }; // NOT ALLOWED. too few template arguments
std::pair<int> p2 { 3, 4 }; // NOT ALLOWED. too few template arguments
```
- Can add literal suffixes to change deduced type
	- `std::pair p1 {1u, 2u};` deduced to pair<unsigned int, unsigned int>

- In C++17, CTAD does not know how to deduce template arguments for aggregate class templates
	- See [[14. Structs, members, member selection]] for what an aggregate is
		- Note: non-aggregates don't need deduction guides in C++17 because of presence of **constructors** serving the same purpose
	- Help it out by giving compiler a **deduction guide**
```cpp
template <typename T, typename U>
struct Pair {
    T first{};
    U second{};
};

// Here's a deduction guide for our Pair (needed in C++17 only)
// Pair objects initialized with arguments of type T and U should deduce to Pair<T, U>
template <typename T, typename U>
Pair(T, U) -> Pair<T, U>;
```

## Type template parameters with default values
eg.
```cpp
template <typename T=int, typename U=int>{ // default T and U to type int
struct Pair
    T first{};
    U second{};
};

template <typename T, typename U>
Pair(T, U) -> Pair<T, U>;
Pair p3;                   // uses default Pair<int, int>
```

## CTAD can't be used in non-static initialisation
- ie. cannot use in **class member declarations**
```cpp
struct Foo {
    std::pair<int, int> p1{ 1, 2 }; // ok, template arguments explicit
    std::pair p2{ 1, 2 };           // compile error, CTAD can't be used in this context
};

int main() {
    std::pair p3{ 1, 2 };           // ok, CTAD can be used here
    return 0;
}
```

## CTAD can't work in function parameters
- A in CTAD is for **argument** deduction, not class template **parameter** deduction
```CPP
void print(std::pair p){ // compile error, CTAD can't be used here
    std::cout << p.first << ' ' << p.second << '\n';
}
```
- Instead should use a template

## Alias templates
```cpp
template <typename T>
using Coord = Pair<T>; // Coord is an alias for Pair<T>

template <typename T>
void print(const Coord<T>& c)
{
    std::cout << c.first << ' ' << c.second << '\n';
}
```

- `Coord<T>` is type alias for `Pair<T>`
- Use `Cood` where we would use `Pair`, `Coord<T>` where we would use `Pair<T>`
- Pre C++20, no template deduction for alias templates

# Class templates with member functions
- Type template parameters defined as part of class template parameter can be used both as type of data members, and as type of member function parameters.
	- Can be defined inside or outside the class
	- Note: **inside**-class template definitions are **implicitly inline**, but if defined **outside**, **not implicitly inline**
		- However, if defined outside class definition, still does not violate ODR because templates follow different rules. No worries.

- Injected class names
	- Why constructor is named `Pair` and not `Pair<T>`?
	- Within the scope of the class, the unqualified name of the class is called **injected class name**, serves as **shorthand for the fully templated name**
		- ie. `Pair` is injected name, if used within `Pair` class scope, refers to `Pair<T>`
	- Notice how this does not apply for member function defined outside the class template

- Notes for defining outside class:
	- Must resupply the template parameter declaration, `template <typename T>`
	- Must give the fully templated name of the class tempalte, `Pair<T>::isEqual`, not just `Pair::isEqual`

> When defining outside class, defining right below! In the same file! 
- Compiler needs to see both class definition and the template member function definition.

```cpp
#include <ios>       // for std::boolalpha
#include <iostream>

template <typename T>
class Pair {
private:
    T m_first{};
    T m_second{};

public:
    // When we define a member function inside the class definition,
    // the template parameter declaration belonging to the class applies
    Pair(const T& first, const T& second)
        : m_first{ first }
        , m_second{ second }
    {
    }

    bool isEqual(const Pair<T>& pair);
};

// When we define a member function outside the class definition,
// we need to resupply a template parameter declaration
template <typename T>
bool Pair<T>::isEqual(const Pair<T>& pair) {
    return m_first == pair.m_first && m_second == pair.m_second;
}

int main() {
    Pair p1{ 5, 6 }; // uses CTAD to infer type Pair<int>
    std::cout << std::boolalpha << "isEqual(5, 6): " << p1.isEqual( Pair{5, 6} ) << '\n';
    std::cout << std::boolalpha << "isEqual(5, 7): " << p1.isEqual( Pair{5, 7} ) << '\n';

    return 0;
}
```
