To get size,

Best practices
- Prefer `int` when the size of the integer doesn’t matter (e.g. the number will always fit within the range of a 2-byte signed integer). For example, if you’re asking the user to enter their age, or counting from 1 to 10, it doesn’t matter whether `int` is 16-bits or 32-bits (the numbers will fit either way). This will cover the vast majority of the cases you’re likely to run across.
- Prefer `std::int#_t` when storing a quantity that needs a guaranteed range.
- Prefer `std::uint#_t` when doing bit manipulation or well-defined wrap-around behavior is required (e.g. for cryptography or random number generation).

Booleans
- To print "true" instead of 1":
	- `cout << std::boolalpha << b << "\n"; // Prints true`
- Conversions
	- Copy initialisation will allow implicit conversion from int to bool.
	- Any non 0 int treated as true.
- cin
	- eg.`cin >> b`
	- If enter 'true', this doesnt work.
	- Non numeric value interpreted as false, failure
	- To do it this way, would need `boolalpha` (which would then only allow 'true' and 'false')
```CPP
std::cin >> std::boolalpha;
std::cin >> b;

//To turn off,
std::cin >> std::noboolalpha;
```


# Type conversion
- Implicit conversion
	- Returns a new temporary object with different data type and different value
- Narrowing conversions 
	- Not allowed by `{}`, hence recommended
- Explicit type conversion
	- `static_cast<new_type> (expression)`

```
char ch {97} //97 is ascii for 'a'
std::cout << ch << " has value " <<static_cast<int>(ch)<<"\n"
```
