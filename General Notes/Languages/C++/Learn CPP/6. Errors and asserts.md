# Common semantic errors
- Conditional logic error
- Infinite loop
- Off by one error
- Indexing error (eg. `s[r]` vs `s[arr[r]]`)
- Incorrect operator precedence
- Floating point precision issues
- Integer division
- Accidental null statements
- Assignment instead of equality 
- Forgetting to use () when calling function

# Handling error
- Handle the error within the function
- Pass the error back to the caller to deal with
- Halt the program
- Throw an exception

# std::cin and handling invalid input
4 types of input error
- Input extraction succeeds but the input is meaningless to the program (e.g. entering ‘k’ as your mathematical operator).
- Input extraction succeeds but the user enters additional input (e.g. entering ‘*q hello’ as your mathematical operator).
- Input extraction fails (e.g. trying to enter ‘q’ into a numeric input).
- Input extraction succeeds but the user overflows a numeric value.

1. Meaningless input - validate input
```CPP
char getOperator()
{
    while (true) // Loop until user enters a valid input
    {
        std::cout << "Enter one of the following: +, -, *, or /: ";
        char operation{};
        std::cin >> operation;

        // Check whether the user entered meaningful input
        switch (operation)
        {
        case '+':
        case '-':
        case '*':
        case '/':
            return operation; // return it to the caller
        default: // otherwise tell the user what went wrong
            std::cout << "Oops, that input is invalid.  Please try again.\n";
        }
    } // and try again
}
```

2. Extraction succeeds but with extraneous input
```CPP
#include <limits> //for numeric_limits
void ignoreLine() {
	std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
}
// Clear all characters out of a buffer, or until a '\n' is removed

bool hasUnextractedInput()
{
    return !std::cin.eof() && std::cin.peek() != '\n';
}

//std::cin.eof(): return true if last input operation reached end of input stream
//std::cin.peek(): peek at next char in input stream without extracting it
```

3. Extraction fails
- cin goes into failure mode, we need to detect, put cin iback in normal mode and remove the bad intput
```CPP
if (!std::cin) // If the previous extraction failed
{
    // Let's handle the failure
    std::cin.clear(); // Put us back in 'normal' operation mode
    ignoreLine();     // And remove the bad input
}
```

4. Overflow
- cin goes into failure mode, but also assigned the closest in-range value to the variable
- Same steps as above

## Checklist
- Check for failed extrction first
- Then check for any unextracted input and remove it
- Then do logical checks


# Asserts
eg.
```CPP
#include <cassert>
assert(gravity > 0.0); 
assert(moved && "Need to handle case where student was just moved to another classroom");
// Use this to document cases. The string literal will evaluate to true
```

Enabling/disabling asserts with NDEBUG
```CPP
#define NDEBUG
#include <cassert>
```

static_assert
- `static_asset(sizeof(long)==8, "long must be 8 bytes);`
- Checked at compile time
	- Values ust be `constexpr` (so can check at compile time)

> Favour static_assert over assert

> Asserts are used to detect programming errors about things that should never happen. If they do, they are programmer's fault
