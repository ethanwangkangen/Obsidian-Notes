# Subnet addressing

- Solution: subnet addressing

### Special addresses
- 0.0.0.0: Network address
	- Identifies the subnet itself
	- Is used in routing tables to represent the entire subnet and is not assignable to any host
	- When used as a **listening address**, it means the service will accept connections from **any network interface**
	- Not rotable
- ???.255: Direct broadcast address
	- Used to send a packet to **every** host on the subnet
	- Is the **last address in any given subnet**, ie. **all HOST BITS set to 1**
	- They are effectively disabled today
- 255.255.255.255: local net broadcast
	- Mainly used for **DHCP**

#### Loopback
- 127.0.0.0/8: Loopback address
	- Virtual interface for host to talk to itself
	- When process sends packet to this address, it goes down network stack, hits loopback interface and comes right back up, **never touches NIC or any physical network**
## Subnet masks
- Assignment of bits used by host/router to determine how to partition the network and subnet info from the host info
	- Configured into a host/router statically, or dynamically eg. through DHCP

![[Pasted image 20260615174623.png]]

- Bit set to 1: corresponding bit is part of **combined network/subnetwork** portion
- Bit set to 0: part of **host** portion

- Note that subnet masking is an **internal** system
	- External routers just go by the network portion and ignore the subnetwork portion
	- eg. Network is 200.10.5.0/24, divided into 4 subnets 200.10.5.0/26,....
		- 2 bits for subnet (26-4), 2^2 = 4 subnetworks
	- External router does not care about the /26 part.

## VLSM (Variable Length Subnet Mask)
- Requirements
	- All routers must run a **classless** routing protocol
	- Classful protocols like RIPv1 don't carry subnet masks in route advertisements, so assume one mask per class and can't do VLSM
- Can allocate subnet sizes to match actual need
- Sort requirements **largest-first**
	- For each requirement, find the smallest block that fits: pick the smallest power of 2 that is ≥ (hosts needed + 2)
	- Allocate that block starting at the next available aligned boundary
- A `/n` block (size 2^(32−n)) must start at an address that's a **multiple of its block size**
    - `/25` (128 addresses) → starts at `.0` or `.128`
    - `/27` (32 addresses) → starts at `.0`, `.32`, `.64`, `.96`, …
    - `/30` (4 addresses) → starts at `.0`, `.4`, `.8`, `.12`, …
- Repeat for the next largest requirement using the remaining space

# Classless Inter-Domain Routing
- To relieve the pressure on the avaialability of IPv4 addresses
- Previously, classes were fixed
	- Address starting with 192.x.x.x was class C so mask was always /24
	- Couldn't get a block of 500 addresses, either got class C (254) or B (65,534) hosts
- Subnetting flexibility
	- CIDR decouples the mask from the prefix
	- 192.51.100.0/22 -> first 22 bits are the network prefix
		- 2^10 = 1024 addresses, 1022 usable

- Where does the mask live?
	- Is not part of the packet itself.
		- In contrast to classful architecture, where routers derived the mask by inspecting the first few bits of address
	- Lives in interfaces and routing tables
	- Travels through **routing protocol messages**

- Supernetting (route aggregation)
	- eg. ISP owns 4 contiguous /24 blocks
	- 200.10.4.0/24
	- 200.10.5.0/24
	- 200.10.6.0/24
	- 200.10.7.0/24
	- In binary the first **22** bits of all 4 are identical
		- So instead of advertising 4 separate routes to the global internet, the ISP advertises one: 20.10.4.0/**22**
# Multicast addresses
- Ip multicast address identifies a group of host interfaces rather than a single one
	- The portion of the network that the single group covers is known as the **scope**
	- eg. node-local(same computer), link-local (same subnet), site-local (applicable to some site), global (entire internet)
- IPv4 multicast address
	- Class D space (224.0.0.0 - 239.255.255.255) reserved for multicast
		- 224.0.0.0/4 - first 4 bits are always 1110, leaving 28 bits to identify the group
	- 28 bits free -> 2^28 host groups (each host group is an IP address)
	- Address is carved into ranges by convention
		- 224.0.0.0/24 - link local multicast
			- 224.0.0.1 - all hosts on this segment
			- 224.0.0.2 - all routers
			- 224.0.0.5 - OSPF routers
		- 224.0.1.0/24 - internetwork control
		- 232.0.0.0/8 - source specific multicast
		- 233.0.0.0/8 - GLOP addressing
		- 239.0.0.0/8 - administratively scoped
