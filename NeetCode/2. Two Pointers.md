## One to iterate from L to R, one to keep track from L to R
- Eg. *Remove Element, Remove Duplicates From Sorted Array*
- Want to keep track of where the next valid index is, to swap with the current iterator
- Relevant for **sorting in place** questions
- 

## Start from opposite ends then move inwards
```cpp
int l {0};
int r = s.size()-1;
while (l < r) {
	    //conditional check
	    l++
        r--;
    }
}
```
- Pairwise, eg. palindromes
- Note : Getting the next unique element:
	- `while (nums[lPtr] == nums[lPtr-1]) lPtr++;`

- Dynamically:
	- Shift left/right pointer inwards depending on which is the limiting factor.
	- Optimisation from brute force, all possibilities seen
	- eg. *Container with most water*
		- Once left pointer shifted inwards, don't have to ever consider any result that includes that left pointer

- When to use < vs <=
	- For pointer changes:
		- `while (l < r)` when each iteration processes exactly one side
		- `while (l <= r)` when every index must be processed individually (binary search etc)
	- For choosing a side:
		- eg. **trapping rain water**, there are 3 cases, 
			- if leftMax<rightMax, if leftMax>rightMax, if leftMax == rightMax
			- The == should only fall under **one of the first 2**
			- Hence, just do
```cpp
if (leftMax <= rightMax) {
    process left
} else {
    process right
}
```
