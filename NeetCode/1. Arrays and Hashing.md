## Hashmap
```cpp
unordered_map<int, int> mp;
return (mp1 == mp2);
mp[key] = value;  
mp[key]++; // auto-initialises to 0
mp.count(key);
mp.find(key);

    cout << p.first << " " << p.second;

```
- Identifiers
	- Frequency of item
	- Or when we want presence of something, but need to **keep track of indices** as well
- If fixed possibilities, eg. alphabets, consider using a **frequency bucket** (fixed size array)
	- eg,`vector<int> count(26, 0);`
	- Get index of alpabet with `c-'a'` (for lower case), `c-'A'` (for upper)
	- Also for frequency count, since count is bounded by the size of the entire array. See bucket sort for *top k frequent elements*
	- Useful for **frequency sort** in O(n) as well!

## Heap
```cpp
// Max heap (default)
priority_queue<int> pq;
pq.push(x); //insert is o(logn)
pq.top(); // o(1)
pq.pop(); // o(logn)

// Min heap
std::priority_queue<int, std::vector<int>, std::greater<int>> pq;
// priority_queue<type, container, comparator>
```
- Identifiers
	- K largest, top K elements, Median, Priority
- But may not be best solution. eg. for *k largest*, best solution is actually **frequency bucket**

## Prefix sum/product/whatever
- When cumulative sum/etc. from 0 to index, or index to end, etc. needed, and useful for calculating value for each index
	- eg. *Subarray Sum Equals k* -> makes use of **hashmap**, **prefix sum** and **one pass technique**
		- All subarrays -> think about all subarrays **ending at each index**
		- For subarray ending at each index i, requires count of previous subarrays that sum up to (prefixSum[i] - k)
		- Can map prefixSums to count, to get the above in O(1).
		- Do it all in a single pass, so at each index I would only have populated the mapping for the **previous** indexes
