# Shortest cycle detection with BFS
- https://leetcode.com/problems/shortest-cycle-in-a-graph/
- This works for a **single node**
	- ie, smallest cycle involving root node.

```cpp
    int bfs(int node, int n, unordered_map<int, vector<int>>& adj, int parent) {
        queue<pair<int, int>> q;
        q.push({node, -1});
        vector<int> dist (n, -1); // distance from root node
        dist[node] = 0;
        int ans = INT_MAX;
        while (!q.empty()) {
            auto [n, p] = q.front();
            q.pop();
            for (auto c : adj[n]) {
                if (c == p) continue;
                if (dist[c] == -1) { // new undiscovered child
                    dist[c] = dist[n] + 1;
                    q.push({c, n});
                } else { // we have seen this child before
                    ans = min(ans, dist[n] + dist[c] + 1);
                }
                
            }
        }
        return ans;
    }
```
- This function returns the shortest cycle **containing the BFS root**, (may overshoot, but its ok since we run for all nodes)
	- To find shortest cycle for entire graph, need to run this **for all nodes**
- For each node in the BFS,
	- For each child
		- Check if unvisited, if so, record distance `dist[node]+1` then push
		- If visited before
			- Cycle distance = the originally found distance of that child from the node `dist[child]` + new distance, which is  (current distance from node `dist[node]` +1)
			- Don't push intot he graph anymore
		- **Don't stop here! Continue checking the whole graph for cycles.**

# Cycle detection with DFS 
## Undirected graph
- Don't even need recursion stack, just keep note of the **parent**
- Don't visit immediate parent.
- If come across a node already visited, for sure have a cycle.
```cpp
bool dfs(int u) {
    visited[u] = true;
    inStack[u] = true;

    for (int v : adj[u]) {

        if (!visited[v]) {
            if (dfs(v))
                return true;
        }
        else if (inStack[v]) {
            return true;  // back edge
        }
    }

    inStack[u] = false;
    return false;
}
```

## Directed graph: Longest cycle in graph (each node max in one cycle)
- https://leetcode.com/problems/longest-cycle-in-a-graph/
- Restriction in question: each node only has **at most one outgoing edge**, which means each node can only be in a single cycle

```cpp
    int dfs(vector<int>& edges, int node, vector<bool>& visited, vector<int>& depths, int currDepth) {
        // check for cycle. if found, immediately return, because each node only involved in one cycle
        if (depths[node] != 0) return currDepth - depths[node];
        // else insert it into the stack
        depths[node] = currDepth;

        // check for child
        int cycle = -1;
        if (edges[node] != -1 and !visited[edges[node]]) {
            cycle = (dfs(edges, edges[node], visited, depths, currDepth+1));
        }
        
        // pop from the stack and visit
        visited[node] = true;
        depths[node] = 0;

        return cycle;
    }
    int longestCycle(vector<int>& edges) {
        int n = edges.size();
        vector<bool> visited (n, false);
        int ans{-1};
        vector<int> depths (n, 0);
        for (int i{}; i < n; ++i) {
            if (!visited[i]) {
                ans = max(ans, dfs(edges, i, visited, depths, 0));
            }
        }
        return ans;
    }
```

## Directed graph: Nodes that lead to cycles/ are in cycles
- https://leetcode.com/problems/find-eventual-safe-states/description/
```cpp
bool dfs(vector<vector<int>>& graph, int node, vector<int>& safe, unordered_set<int>& state) {
        if (state.contains(node)) { // unsafe
            safe[node] = 2;
            return true;
        }

        // add to state
        state.insert(node);

        bool isSafe = true;
        for (int child : graph[node]) {
            if (safe[child] == 2 or (safe[child] == 0 and dfs(graph, child, safe, state))) isSafe = false;
        }

        // remove from state
        state.erase(node);

        // node is safe
        safe[node] = isSafe ? 1 : 2;
        return !isSafe;
    }
    vector<int> eventualSafeNodes(vector<vector<int>>& graph) {
        // node is safe if no cycle involved.

        //adjacency list is already given.
        int n = graph.size();

        vector<int> safe (n, 0); // 0 for unvisited, 1 for safe, 2 for unsafe
        for (int i{}; i < n; ++i) {
            if (safe[i] == 0) { // unvisited
                unordered_set<int> state;
                dfs(graph, i, safe, state);
            }
        }
        vector<int> ans;

        for (int i{}; i < n; ++i) {
            if (safe[i] == 1) ans.push_back(i);
        }
        return ans;
        
    }
```

## Common points among the above
- In the DFS, keep track of the running stack of nodes
	- Either store the `depth[node]`, which can be used to calculate cycle length, or just the presence of it in the stack (unordered_set)
	- Check if node is already in the stack
	- DFS for children
	- After that, remove it from the stack/reset the depth.
	- Visit the node.
	- Then return safe/not safe, cycle or no, etc.
- Difference between node **visited** and **in stack**
	- **visited** -> We already know if the node is in a cycle or not, don't need to check it again
		- Can only do this at the **end** of a DFS function.
		- Normally, we can do this while checking children. If child isn't visited, visit and mark visited. **BUT WE CAN"T DO THIS WHEN CHECKING FOR CYCLES!!!!** Again, only do it for the **current node**, at the **end of the DFS function**, since 'visited' =  we know the answer for this node and don't ever have to check it again
	- **in stack** -> node is currently on the active DFS path. An edge to it immediately proves a cycle.
		- Set at **entry**
		- Cleared **before return**
